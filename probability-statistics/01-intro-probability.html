Introduction to
Probability

Probability is just a number between 0 and 1 that measures how likely something is. Master the language here and every later topic — distributions, Bayes, testing — follows.

2 Why this topic exists

The world is uncertain — a coin, a customer's choice, a measurement error. Probability gives us a number for "how likely" so we can reason about uncertainty instead of guessing.

It's the foundation of all of statistics and machine learning: every model that estimates, predicts, or is "confident" is built on the rules you learn here.

4 Basic definitions
  • Random experiment: a process with an uncertain outcome (a die roll).
  • Sample space $S$: the set of all possible outcomes. Outcome: one element of $S$.
  • Event: a subset of $S$ (a collection of outcomes). $P(A)$: the probability of event $A$.
  • Equally likely: every outcome has the same chance (a fair die/coin).
  • Complement $A^c$: "$A$ does not happen". Mutually exclusive: $A$ and $B$ can't both happen ($A\cap B=\varnothing$). Exhaustive: events that together cover all of $S$.
📖 Study in Depth — probability from zero, one idea per card

Probability is the maths of uncertainty — it gives every event a number between 0 (impossible) and 1 (certain). It is the foundation for random variables, distributions, Bayes' theorem, expectation, sampling and all of statistics and ML. Open each card in order.

1. Why probability exists

Real life is full of uncertainty — will it rain, will a user click an ad, will a model predict correctly? Probability puts a number between 0 and 1 on how likely something is.

0   = impossible
1   = certain
0–1 = possible but uncertain   (fair coin head = 0.5 = 50%)
2. Game imagination

Think of a loot box that can drop a common, rare, epic or legendary item. Probability is the chance meter:

legendary drop = 0.01  →  1% chance

Drop rates, critical-hit chance, spawn chance — all are probability.

3. Basic definition 
P(E) = (favorable outcomes) / (total possible outcomes)
fair die, even number: favorable {2,4,6}=3, total 6 → P = 3/6 = 1/2
4. The probability range 
0 ≤ P(E) ≤ 1
0 impossible · 0.5 even chance · 1 certain
P = −0.2 or 1.5 are IMPOSSIBLE values

GATE trap: probability is never negative and never above 1.

5. Experiment, outcome, sample space, event 
Experiment  = a process with an uncertain result (roll a die)
Outcome     = one possible result (a 4)
Sample space S = the set of ALL outcomes:  S = {1,2,3,4,5,6}
Event E     = a subset of S, e.g. "even" → E = {2,4,6} ⊂ S
6. Sample point, simple & compound events

More precise vocabulary for the "world" of a problem:

random experiment = a process whose result is uncertain (but the
                    set of possible results is known)
sample point      = ONE outcome (a single 'H', or a '4')
finite sample space = a countable number of outcomes

By size, events are either:

simple event   = one sample point      e.g. {4}
compound event = many sample points    e.g. even = {2,4,6}
7. Dice example 
S = {1,2,3,4,5,6}
E = "greater than 4" = {5,6}
P(E) = 2/6 = 1/3
8. Equally likely outcomes

The formula favorable/total works only when outcomes are equally likely (a fair coin/die).

If a coin is biased, P(H) ≠ 1/2. Memory: classical probability assumes equally likely outcomes.

9. Events as sets: ∪, ∩, complement

Events are subsets of the sample space S, so set notation describes them:

A ∪ B  = A OR B happens
A ∩ B  = A AND B happen
Aᶜ     = NOT A (complement)
∅      = the impossible event (empty set)
die: A = even = {2,4,6}, B = ">4" = {5,6}
A ∩ B = {6} ,  A ∪ B = {2,4,5,6} ,  Aᶜ = {1,3,5}
10. The three probability axioms

Every probability must obey three rules — the "constitution" of probability:

1. Non-negativity:  P(A) ≥ 0
2. Normalization:   P(S) = 1
3. Additivity:      if A ∩ B = ∅ then P(A ∪ B) = P(A) + P(B)

From these it follows that 0 ≤ P(A) ≤ 1 for every event.

11. Impossible & certain events 
P(∅) = 0   impossible event (e.g. rolling a 7 on a 6-sided die)
P(S) = 1   certain event   (e.g. rolling a number < 7)

Trap: impossible (P = 0) is not the same as "very unlikely" (P ≈ 0); certain (P = 1) is not the same as "very likely" (P = 0.99).

12. Mutually exclusive events & the addition rule

Mutually exclusive = cannot happen together: A ∩ B = ∅. Then the axiom gives:

P(A ∪ B) = P(A) + P(B)
card: P(king or queen) = 4/52 + 4/52 = 2/13
die:  P(1 or 3 or 5) = 1/6 ×3 = 1/2

Trap: for mutually exclusive events the sum can't exceed 1 — so P(A)=0.7, P(B)=0.5 mutually exclusive is impossible.

13. The complement rule 
P(Aᶜ) = 1 − P(A)        (since A ∪ Aᶜ = S and A ∩ Aᶜ = ∅)
P(not 6) = 1 − 1/6 = 5/6
P(rain) = 0.3 → P(no rain) = 0.7

The fastest tool for "not" and "at least one" problems — count the opposite and subtract from 1.

14. The general addition rule (with overlap)

When A and B can overlap, subtract the double-counted part:

P(A ∪ B) = P(A) + P(B) − P(A ∩ B)
die: even {2,4,6} OR ">3" {4,5,6}, overlap {4,6}
P = 3/6 + 3/6 − 2/6 = 4/6 = 2/3

It reduces to P(A)+P(B) exactly when A, B are mutually exclusive (overlap = 0).

15. Neither, only-A, and finding a missing value

From the addition rule you can pull out every piece of two events:

neither A nor B = (A ∪ B)ᶜ  →  1 − P(A ∪ B)
only A          = P(A) − P(A ∩ B)
only B          = P(B) − P(A ∩ B)
only A + only B + both + neither = 1

And rearrange the addition rule to solve for any unknown:

P(A∩B) = P(A) + P(B) − P(A∪B)
P(A)=0.6, P(B)=0.5, P(A∩B)=0.2 →
only A=0.4, only B=0.3, union=0.9, neither=0.1
16. Venn-diagram view of the addition rule

Picture probability as area: two overlapping circles A and B.

overlap        = A ∩ B
both circles   = A ∪ B
outside both   = neither
only A + only B + overlap = A ∪ B

Adding P(A) + P(B) counts the overlap twice, so you subtract it once — that is exactly P(A∪B) = P(A)+P(B)−P(A∩B).

17. Mutually exclusive vs independent (key trap) 
Mutually exclusive:  A ∩ B = ∅   →  P(A ∩ B) = 0   (can't both happen)
Independent:         one doesn't affect the other → P(A ∩ B) = P(A)·P(B)

These are different ideas. Two non-trivial mutually exclusive events are actually dependent: if A happens, B becomes impossible — so A affects B.

GATE trap: don't write P(A∩B)=P(A)P(B) for mutually exclusive events — there P(A∩B)=0.

18. Exhaustive events & partitions

Exhaustive events together cover the whole sample space:

odd {1,3,5} ∪ even {2,4,6} = S  → exhaustive

Events that are both mutually exclusive AND exhaustive form a partition of S (no overlap, nothing left out).

Why it matters: partitions are the basis of the total-probability rule and Bayes' theorem (next topic).

19. Mutually exclusive vs exhaustive — the four cases

These are two independent properties — don't confuse them:

mutually exclusive = no overlap   (A ∩ B = ∅)
exhaustive         = cover all of S (A ∪ B = S)

So any pair of events falls into one of four cases (die, S = {1..6}):

ME but NOT exhaustive:   {1}        {2}            (no overlap, miss 3–6)
exhaustive but NOT ME:   {1,2,3,4}  {3,4,5,6}      (cover S, overlap {3,4})
BOTH (a partition):      {1,2}  {3,4}  {5,6}       (no overlap, cover S)
NEITHER:                 {1,2}      {2,3}          (overlap, miss 4–6)

Always true: an event and its complement A, Aᶜ are both mutually exclusive (A∩Aᶜ=∅) and exhaustive (A∪Aᶜ=S).

20. "At least", "exactly", "at most"

Read the wording carefully — it changes the favorable set.

exactly one = one and only one
at least one = one or more
at most one  = zero or one
two coins, counting heads:
exactly one  → {HT, TH}        P = 2/4
at least one → {HH, HT, TH}    P = 3/4
at most one  → {HT, TH, TT}    P = 3/4

Tip: "at least one" is usually fastest via the complement: 1 − P(none).

21. Two dice & ordered outcomes

Two dice have 6 × 6 = 36 equally likely ordered pairs (a, b) — and (2,5) ≠ (5,2).

P(sum = 7): {(1,6),(2,5),(3,4),(4,3),(5,2),(6,1)} → 6/36 = 1/6

Same caution for drawing 2 cards: ordered draw = 52 × 51, but an unordered hand = 52C2 — always check whether order matters.

22. Counting: the addition rule (OR)

If a task can happen one way OR another (no overlap), add the counts.

choose 3 red balls OR 4 blue balls → 3 + 4 = 7 choices

OR (mutually exclusive) → add.

23. Counting: the multiplication rule (AND)

If a task has several steps done together, multiply.

3 shirts AND 2 pants → 3 × 2 = 6 outfits

AND (steps in sequence) → multiply.

24. Independent events multiply 
independent A, B:  P(A and B) = P(A∩B) = P(A)·P(B)
two fair coins: P(HH) = 1/2 × 1/2 = 1/4
25. What 'independent' means

Independent = one event does not affect the other. Coin toss 1 doesn't change toss 2; rolling a die twice — the rolls are independent.

P(6 then 6) = 1/6 × 1/6 = 1/36
26. With vs without replacement

Bag: 3 red, 2 blue (total 5).

WITH replacement (put it back): probabilities stay the same
 P(red, red) = 3/5 × 3/5 = 9/25

WITHOUT replacement (keep it): probabilities change
 P(red, red) = 3/5 × 2/4 = 3/10

GATE trap: with replacement → same; without → the second probability changes.

27. Factorial 
n! = n × (n−1) × … × 1
5! = 120,  4! = 24,  3! = 6,  1! = 1,  0! = 1

0! = 1 by convention, so the counting formulas work.

28. Permutation (order matters)

An arrangement — order counts.

nPr = n! / (n−r)!
1st & 2nd prize from 5 people: 5P2 = 5!/3! = 5×4 = 20
29. Combination (order doesn't matter)

A selection — order doesn't count ({A,B} = {B,A}).

nCr = n! / (r!(n−r)!)
team of 2 from 5: 5C2 = 10

Memory: Permutation = positions/order; Combination = group only.

30. Counting by slots (no formula)

You can count arrangements by filling positions one at a time — often faster than a formula.

5 players, rank 1st & 2nd:
position 1 → 5 choices
position 2 → 4 choices
total = 5 × 4 = 20   (= 5P2)
31. Permutation ↔ combination (nPr = nCr · r!)

A permutation is "choose, then arrange":

nPr = nCr × r!      ⇒      nCr = nPr / r!

Combination divides by r! because the r! different orderings of the same group all count as ONE selection.

5P2 = 20 ,  r! = 2!  →  5C2 = 20/2 = 10
32. Combination shortcuts & symmetry 
nC0 = 1 ,  nC1 = n ,  nCn = 1 ,  nC(n−1) = n

Symmetry: nCr = nC(n−r) — choosing r to include = choosing (n−r) to exclude.

100C98 = 100C2  (much easier to compute)
33. Arranging repeated letters

With repeats, divide by the factorial of each repeat count:

arrangements = n! / (p! q! r! …)
MOM:   3!/2!     = 3
BALL:  4!/2!     = 12
LEVEL: 5!/(2!2!) = 30   (L×2, E×2)

Trap: forgetting to divide by repeats overcounts.

34. Counting problems: committees, codes, seating

Committee with exactly 2 boys + 1 girl (6 boys, 4 girls):

choose boys × choose girls = 6C2 × 4C1 = 15 × 4 = 60   (AND → multiply)

3-digit code from 0–9:

repetition allowed:     10 × 10 × 10 = 1000
repetition not allowed: 10 × 9 × 8   = 720  (= 10P3)

Seating:

seat 4 people in 4 chairs: 4! = 24
seat 4 of 10 in 4 chairs:  10P4 = 5040
35. Deciding: does order matter?

First question on any counting problem: does order matter? Yes → permutation; No → combination. Then check: distinct or repeated items? replacement allowed? restrictions?

Word in the questionUsually means
arrange, rank, order, seat, password, number platepermutation
choose, select, committee, team, group, hand of cardscombination

Memory: P = Position (order); C = Choose (group). And 10P3 = 720 ≠ 10C3 = 120.

36. Counting → probability examples

Bag of 5 red + 3 blue, pick one:

P(red) = 5/8

Standard 52-card deck:

P(ace)   = 4/52 = 1/13
P(heart) = 13/52 = 1/4
37. Common GATE traps
  • Probability > 1? No — always 0 ≤ P ≤ 1.
  • Independent: P(A and B) = P(A)+P(B)? No — it's P(A)·P(B).
  • With vs without replacement give the same answer? No — without replacement changes the probabilities.
  • Permutation = combination? No — order matters vs doesn't.

39. Cheatsheet
  • P(E) = favorable/total;   0 ≤ P ≤ 1; sample space = all outcomes; event ⊂ S.
  • OR (exclusive) → add; AND (steps) → multiply.
  • Independent: P(A∩B) = P(A)P(B).
  • With replacement → unchanged; without → changes.
  • nPr = n!/(n−r)! (order matters); nCr = n!/(r!(n−r)!) (order doesn't).
40. Memory box & mental model
  • Probability = a chance meter from 0 to 1.
  • Sample space = every outcome; event = the outcomes you care about.
  • Independent events multiply; OR adds, AND multiplies.
  • Permutation = order matters; combination = order doesn't.

6 Main formulas

Classical (equally likely) probability:

$$ P(A)=\frac{\text{favorable outcomes}}{\text{total outcomes}}=\frac{|A|}{|S|} $$

Axioms (Kolmogorov): $0\le P(A)\le 1$,   $P(S)=1$,   and for disjoint events probabilities add.

$$ P(A^c)=1-P(A) \qquad P(A\cup B)=P(A)+P(B)-P(A\cap B) $$

If $A$ and $B$ are mutually exclusive, $P(A\cap B)=0$, so $P(A\cup B)=P(A)+P(B)$.

Counting (how to get $|A|$ and $|S|$):

$$ ^nP_r=\frac{n!}{(n-r)!}\quad(\text{order matters}) \qquad ^nC_r=\binom{n}{r}=\frac{n!}{r!\,(n-r)!}\quad(\text{order doesn't}) $$

Multiplication principle: if step 1 has $m$ ways and step 2 has $n$ ways, together they have $m\times n$ ways.

7 Step-by-step method
  1. Write the sample space $S$ and count its size $|S|$.
  2. Describe the event $A$ as a set of outcomes; count $|A|$.
  3. For a single event use $P(A)=|A|/|S|$.
  4. For "$A$ or $B$", use the addition rule (subtract the overlap).
  5. For "at least one", it's often easier to use the complement: $1-P(\text{none})$.
8 Worked examples
✦ One die

$S=\{1,2,3,4,5,6\}$, so $|S|=6$. Event "even" $=\{2,4,6\}$ → $P=\tfrac{3}{6}=\tfrac12$. Event "greater than 4" $=\{5,6\}$ → $P=\tfrac{2}{6}=\tfrac13$.

✦ Two dice — sum equals 7

$|S|=36$. Favorable: $(1,6),(2,5),(3,4),(4,3),(5,2),(6,1)$ → 6 ways. $P=\tfrac{6}{36}=\tfrac16$.

✦ Addition rule

One card from 52: $P(\text{King or Heart})=P(K)+P(H)-P(K\cap H)=\tfrac{4}{52}+\tfrac{13}{52}-\tfrac{1}{52}=\tfrac{16}{52}=\tfrac{4}{13}$.

9 GATE cheat sheet
⚡ High-value facts
  • Equally likely ⇒ probability is just counting: favorable ÷ total.
  • "At least one" → use the complement: $1-P(\text{none})$ (huge time-saver).
  • Addition rule always subtracts the overlap: $P(A\cup B)=P(A)+P(B)-P(A\cap B)$.
  • All outcome probabilities sum to $1$; a valid probability is always in $[0,1]$.
  • Mutually exclusive means they can't co-occur — not the same as independent.
  • Order matters → permutation $^nP_r$; order doesn't → combination $^nC_r$ (always $^nC_r\le{}^nP_r$).
  • Multiplication principle: independent choices multiply ($m\times n$).
10 Common traps
✕ Watch out

Trap: "Mutually exclusive = independent." Reality: opposite, really — if disjoint events both have $P\gt 0$, they are dependent (one happening rules the other out).

Trap: "$P(A\cup B)=P(A)+P(B)$ always." Reality: only if disjoint; otherwise subtract $P(A\cap B)$.

Trap: "Probability can exceed 1." Reality: never — $0\le P\le 1$.

Trap: "Use permutations to pick a committee." Reality: a committee is unordered — use combinations $^nC_r$; permutations over-count by $r!$.

11 Connections with other topics
  • Conditional & Bayes: updating probability once you know extra information.
  • Random variables: attach numbers to outcomes so we can average them.
  • Distributions: the full list of probabilities over all outcomes.
  • Statistics: probability run in reverse — infer the unknown chance from data.
12 Practice problems

Q1. A fair die is rolled. $P(\text{odd})=?$

Odd $=\{1,3,5\}$ → $\tfrac{3}{6}=\tfrac12$.

Q2. Two fair coins. $P(\text{at least one head})=?$

Use complement: $1-P(\text{no heads})=1-\tfrac14=\tfrac34$.

Q3. $P(A)=0.5,\ P(B)=0.4,\ P(A\cap B)=0.2$. Find $P(A\cup B)$.

$0.5+0.4-0.2=0.7$.

Q4. Two dice. $P(\text{sum}=7)$ vs $P(\text{sum}=2)$ — which is larger?

Sum 7 has 6 ways ($\tfrac{6}{36}$); sum 2 has 1 way ($\tfrac{1}{36}$). Sum 7 is larger.

Q5. If $P(A)=0.3$, what is $P(A^c)$?

$1-0.3=0.7$.

Q6. From 5 people, how many ways to choose a committee of 2?

Unordered: $^5C_2=\dfrac{5!}{2!\,3!}=10$.

Q7. From 5 people, how many ways to choose a president then a VP?

Ordered: $^5P_2=5\times4=20$ (twice $^5C_2$ because roles differ).

Quick MCQ

For events with $P(A)=0.6,\ P(B)=0.5,\ P(A\cap B)=0.3$, what is $P(A\cup B)$?
That's $P(A)+P(B)$ — you forgot to subtract the overlap.
That's just the intersection.
✅ Correct — $0.6+0.5-0.3=0.8$.
Close, but subtract the full overlap $0.3$, giving $0.8$.
