Types of Optimization

Optimization means finding the best value — the lowest (or highest) point of a function. First, learn the vocabulary and the kinds of problems.

2 Why this topic exists

Almost every model "learns" by minimizing something — a loss, an error, a cost. Before we compute minima, we need the language: what's the objective, what can we change, are there restrictions, and is the landscape friendly (convex) or full of traps (non-convex)?

3 Game / visual imagination
🎮 Imagine it

The objective function is a landscape of hills and valleys. Optimization = find the lowest valley (a minimum). A convex problem is a single smooth bowl — wherever you are, roll downhill and you reach the bottom. A non-convex one has many valleys, so you can get stuck in a local one that isn't the deepest.

🧪 Convex vs Non-convex one global min vs many valleys
4 Basic definitions
  • Objective (cost) function $f(x)$: what we minimize or maximize. Decision variables $x$: what we are free to change.
  • Constraints: conditions $x$ must satisfy. Feasible region: the set of all $x$ that satisfy them.
  • Unconstrained vs constrained: $x$ free to be anything, vs $x$ limited by constraints (e.g. $x\ge0$).
  • Global minimum: the lowest value anywhere. Local minimum: lowest only within a small neighbourhood.
  • Convex function: a bowl shape — the chord between any two points stays above the curve. Convex set: the segment between any two feasible points stays feasible.
  • Why convexity matters: for a convex problem, every local minimum is the global minimum.
  • Linear vs nonlinear: the objective and constraints are straight-line (LP) or curved (NLP).
  • Discrete vs continuous: variables take whole-number / categorical values vs any real value.
📖 Study in Depth — types of optimization & the vocabulary, from zero
1. What optimization means

Optimization = find the input that makes an objective as good as possible (smallest cost, or largest reward).

minimize (or maximize) f(x)   by choosing the decision variables x

In data science the objective is usually a loss we minimize (training a model = optimization).

2. Standard form: everything is a minimization 
min over x   f(x)      subject to (optional) constraints

Maximizing is just minimizing the negative, so one toolkit covers both:

max f(x) = − min ( −f(x) )
3. Game imagination — find the lowest valley

Picture the objective as a landscape of hills and valleys. Optimization is releasing a ball and letting it settle in the lowest valley.

height of land = f(x) · ball's resting spot = the minimum

Constraints = fences that keep the ball inside an allowed region.

4. The four ingredients
  • Objective function f(x) — what we score
  • Decision variables x — what we control
  • Constraints — rules x must obey (e.g. x ≥ 0, g(x) ≤ 0)
  • Feasible region — the set of all x that satisfy every constraint

The answer is the best feasible point.

5. Global vs local optima 
global minimum = lowest value anywhere
local minimum  = lowest only within a small neighbourhood

A landscape can have many local valleys but only one true global bottom. Methods can get stuck in a local minimum — a central difficulty in non-convex problems.

6. Unconstrained vs constrained 
unconstrained: x can be anything (just set the derivative/gradient to 0)
constrained:   x must satisfy equalities/inequalities (use Lagrange / KKT)
min x²            → unconstrained
min x²  s.t. x ≥ 2 → constrained (answer sits on the boundary x = 2)
7. Single-variable vs multi-variable 
univariate:   f(x), one knob        → solve f′(x) = 0
multivariate: f(x₁,…,xₙ)            → solve ∇f = 0

These are the next two topics; the idea generalizes from a derivative to a gradient.

8. Continuous vs discrete 
continuous: x is a real number (heights, weights, model weights)
discrete:   x takes whole/finite values → integer / combinatorial optimization

Discrete problems (routes, schedules, knapsack) can't use derivatives and are often much harder.

9. Linear vs nonlinear programming GATE extra 
Linear Programming (LP):    objective & constraints all linear → optimum at a corner of the feasible polygon
Nonlinear Programming (NLP): objective or a constraint is curved

LP is solved by methods like simplex; its optimum always lies at a vertex of the feasible region.

10. Convex vs non-convex — the key divide

A convex problem (bowl-shaped objective over a convex region) has the wonderful property:

any local minimum is the GLOBAL minimum

So convex problems (linear/logistic regression, SVM, ridge/lasso) are reliably solvable. Non-convex ones (neural nets) have many local minima/saddles — no such guarantee.

11. Analytical vs iterative; deterministic vs stochastic 
analytical: solve f′=0 / ∇f=0 in closed form (small, clean problems)
iterative:  step toward the optimum (gradient descent) when no closed form exists
deterministic: same path every run · stochastic (SGD): random samples, noisy path
12. Where optimization shows up in DA/ML
  • training any model = minimize a loss (least squares, cross-entropy)
  • regularization adds a penalty to the objective (ridge/lasso)
  • SVM = constrained convex optimization; PCA = maximize variance
  • resource/portfolio/scheduling problems = LP / NLP
13. Common GATE traps
  • Local minimum = global minimum? Only for convex problems.
  • Maximization needs different methods? No — maximize f = minimize −f.
  • LP optimum is inside the region? No — it's at a corner/vertex.
  • All optimization uses derivatives? No — discrete/combinatorial doesn't.

6 Main idea

Standard form — minimize the objective, optionally subject to constraints:

$$ \min_{x} \ f(x) \qquad \text{s.t. (optional) constraints} $$

Max = flipped min — maximizing is minimizing the negative:

$$ \max_x f(x) = -\min_x\big(-f(x)\big) $$

So every minimization method also handles maximization — just flip the sign.

7 Step-by-step: classify a problem
  1. What is the objective $f(x)$, and are we minimizing or maximizing?
  2. What are the decision variables — one ($\mathbb{R}$) or many ($\mathbb{R}^n$)?
  3. Any constraints? If none → unconstrained.
  4. Is $f$ convex? If yes, any local minimum is the global minimum.
8 Worked example
✦ Classify $\min (x-3)^2$

Objective $f(x)=(x-3)^2$; one variable; no constraints → unconstrained univariate.

$f$ is a parabola opening up → convex, single bowl. The minimum is at $x^*=3$ with $f=0$, and it's global.

9 GATE cheat sheet
⚡ High-value facts
  • Maximize $f$ = minimize $-f$ (flip the sign).
  • Convex ⇒ every local minimum is global (no traps).
  • A sum of convex functions is convex; affine (linear) functions are convex.
  • Linear objective + linear constraints = a Linear Program (LP).
  • Unconstrained means the optimum is found purely from derivatives/gradients.
10 Common traps
✕ Watch out

Trap: "Local minimum = global minimum." Reality: true only when $f$ is convex; otherwise you can be stuck locally.

Trap: "Convex function and convex set are the same." Reality: one is about the curve's shape, the other about the feasible region.

Trap: "Maximizing needs a different method." Reality: just minimize $-f$.

11 Connections with other topics
  • Univariate / multivariate: the methods that actually find these minima.
  • Gradient descent: a general algorithm that works even when calculus is hard.
  • Linear algebra: the Hessian (a matrix) decides if a point is a min, max, or saddle.
  • Machine learning: training = minimizing a (often non-convex) loss function.
12 Practice problems

Q1. Is $f(x)=x^2$ convex or non-convex?

Convex — an upward parabola (single bowl).

Q2. Rewrite "maximize $g(x)$" as a minimization.

Minimize $-g(x)$.

Q3. For a convex $f$, is a local minimum guaranteed to be global?

Yes.

Q4. "$\min f(x)$ subject to $x\ge0$" is constrained or unconstrained?

Constrained (there's a restriction on $x$).

Quick MCQ

For a convex function, a local minimum is…
Opposite — convexity guarantees it.
For convex $f$ it's always global, not sometimes.
✅ Correct — convexity means no local-minimum traps; local = global.
A minimum isn't a maximum.

🧠 Remember

  • $\max f=-\min(-f)$.
  • Convex ⇒ local min is global (single bowl, no traps).
  • Unconstrained = free $x$; constrained = $x$ must satisfy conditions.
