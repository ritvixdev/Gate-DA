Topic 4 · GATE weight: Very High Core

Logistic Regression

Despite the name, it's a classifier. It squashes a linear score through an S-curve to output a probability between 0 and 1.

2 Why this topic exists

Linear regression can output any number — useless for "yes/no". Logistic regression feeds the linear score into the sigmoid, turning it into a probability. It's the standard, interpretable baseline for binary classification.

3 Game / visual imagination
🎮 Imagine it

Take the straight-line score $z=\beta_0+\beta_1 x$ and pour it through an S-shaped funnel (the sigmoid): very negative $z$ → near 0, very positive → near 1, $z=0$ → exactly 0.5. The decision boundary is where the probability crosses 0.5 (i.e. $z=0$).

4 Basic definitions
  • Sigmoid $\sigma(z)$: the S-curve squashing any real number into a probability in $(0,1)$.
  • Linear score (logit) $z$: $\beta_0+\beta_1 x_1+\dots$ — the input fed to the sigmoid.
  • Predicted probability $\hat{p}$: $\sigma(z)$, the model's estimate of $P(y=1)$.
  • Odds & log-odds: odds $=p/(1-p)$; the log-odds equal $z$ (linear in the features).
  • Decision boundary: where $\hat{p}=0.5$ (i.e. $z=0$); predict 1 when $\hat{p}\ge0.5$.
  • Threshold: the cutoff (default 0.5) you compare $\hat{p}$ against — move it to trade precision vs recall.
  • Log-loss (cross-entropy): the cost function logistic regression minimizes.
  • Maximum likelihood: the fitting principle — choose $\beta$ making the observed labels most probable.
📖 Study in Depth — classification & the full neural/deep-learning stack (CNN, RNN, embeddings, attention)
1. Classification — predicting a category

Classification is the supervised task where the target is a discrete class, not a number:

spam / not spam · cat / dog · fraud / not · rain / no-rain · disease / healthy

Logistic regression (this topic) is the workhorse binary classifier — first the general picture, then its sigmoid mechanics.

2. Class labels & discrete targets

The output Y is one of a fixed set of class labels, usually coded as integers:

spam: 0 = not spam, 1 = spam
sentiment: 0 = negative, 1 = neutral, 2 = positive

The codes are just names — for unordered classes their numeric size is meaningless.

3. Binary classification

Exactly two classes, coded 0 / 1:

spam/not · fraud/not · churn/not · pass/fail · disease/healthy

Logistic regression natively solves the binary case.

4. Multi-class classification

More than two classes:

digits 0–9 · cat/dog/car/bird · sports/politics/tech/business · blood group A/B/AB/O

Built from binary classifiers (one-vs-rest) or a softmax output.

5. Categorical variables need encoding

Most algorithms need numbers, so category inputs must be encoded.

smoker: yes→1, no→0
risk (ordinal): low→0, medium→1, high→2   ✓ order is real
6. Ordinal vs one-hot — the fake-order trap

For unordered (nominal) categories, integer codes invent a false ranking:

Delhi=0, Kolkata=1, Bangalore=2  ⇒ implies Bangalore > Kolkata?? (nonsense)

Use one-hot encoding instead — one 0/1 column per category:

Delhi → (1,0,0) · Kolkata → (0,1,0) · Bangalore → (0,0,1)

Ordinal data → integer codes; nominal data → one-hot.

7. Class probability P(class | input)

Most classifiers output probabilities, not just a label:

P(spam | email) = 0.92 ,  P(not spam | email) = 0.08
P(cat)=0.70, P(dog)=0.20, P(car)=0.10   (sum to 1)

Logistic regression's sigmoid produces exactly this probability.

8. Threshold — turning probability into a label

For binary problems, compare the probability to a threshold (default 0.5):

if P(class=1) ≥ 0.5 → predict 1, else 0
P(churn)=0.72 → churn ; P(churn)=0.35 → not churn

The threshold is tunable: for fraud you might use 0.3 — missing a fraud is costlier than a false alarm.

9. Multi-class: pick the argmax 
Cat 0.10 · Dog 0.75 · Car 0.15 → predict Dog

argmax = choose the class with the highest probability. (Thresholds are mainly a binary tool.)

  • Logistic Regression · Naive Bayes · k-Nearest Neighbors
  • Decision Tree · Random Forest · SVM · Neural Network

GATE-favourites: logistic regression, kNN, Naive Bayes, decision-tree basics.

12. Imbalanced data & the accuracy trap

When one class is rare, plain accuracy lies:

99% transactions normal, 1% fraud
"always predict not-fraud" → 99% accuracy, 0 frauds caught!

So for imbalanced problems use precision, recall, F1 — not accuracy alone.

13. Evaluation metrics — a preview 
accuracy = correct / total
  • precision — of predicted-positives, how many were right
  • recall (sensitivity) — of actual-positives, how many we caught
  • F1 — harmonic mean of precision & recall; ROC-AUC — ranking quality

Medical diagnosis prizes recall (a missed disease is dangerous); spam filters prize precision.

14. Where classification is used
  • spam detection, fraud detection, churn prediction
  • image / object recognition, face recognition (multi-class)
  • medical diagnosis, sentiment analysis, webpage categorization
  • speech recognition, click/buy prediction for recommendations
15. Why logistic regression (not linear)

Linear regression can output any number (−2, 0.4, 5…), but a probability must lie in [0, 1]. Logistic regression fixes this by squashing the linear score through a sigmoid.

"regression" in the name, but it is a classification algorithm
16. The sigmoid function 
z = wᵀx + b
σ(z) = 1 / (1 + e^(−z))            P(Y=1 | X) = σ(wᵀx + b)
z → +∞ ⇒ σ → 1 · z = 0 ⇒ σ = 0.5 · z → −∞ ⇒ σ → 0

The sigmoid maps any real number into (0, 1) — a valid probability.

17. The S-curve & effect of w, b

Sigmoid is an S-shape: flat near 0 and 1, steep in the middle.

w sign → direction (w>0: P rises with x; w<0: P falls)
|w| large → steeper transition
b → shifts the curve left/right
18. From features to a label 
X → z = wᵀx + b → p = σ(z) → threshold → class 0/1
tumor: P(malignant)=0.91 ≥ 0.5 → malignant ;  0.22 < 0.5 → benign

Default threshold 0.5, but lower it when misses are costly (medical screening).

19. Why RSS is wrong for classification

Squared error doesn't punish a confident wrong probability the way classification needs:

y=1, p=0.9 → good ;  y=1, p=0.1 → very bad, but (1−p)² barely differs

So classification uses cross-entropy / log loss instead of RSS.

20. Cross-entropy (log loss) 
Loss = −[ y·log(p) + (1−y)·log(1−p) ]
y = 1 → −log(p)        y = 0 → −log(1−p)

It is exactly the cost logistic regression minimizes during training.

21. Why confident-wrong is punished hard 
y=1, p=0.9   → −log(0.9)  ≈ 0.105   (small)
y=1, p=0.1   → −log(0.1)  ≈ 2.303   (large)
y=1, p=0.001 → −log(0.001)≈ 6.907   (huge)

The more confident and wrong the model is, the steeper the penalty — pushing it toward honest probabilities.

22. Naive Bayes — a Bayes-theorem classifier GATE extra

Naive Bayes classifies using Bayes' theorem; it's "naive" because it assumes features are conditionally independent given the class.

P(C | X) = P(X | C)·P(C) / P(X)
posterior = likelihood × prior / evidence  →  pick the class with the largest posterior
23. Prior, likelihood, evidence, posterior GATE extra
  • Prior P(C): belief before seeing features (P(spam)=0.3).
  • Likelihood P(X|C): chance of the features in that class (P("win"|spam) high).
  • Evidence P(X): same for all classes, so dropped when comparing.
  • Posterior P(C|X): updated belief after the features.
P(C | X) ∝ P(X | C) · P(C)
24. The naive independence assumption GATE extra

Assume each feature contributes independently given the class, so the likelihood factorizes:

P(X | C) = P(x₁|C)·P(x₂|C)·…·P(xₙ|C)
⇒ P(C | X) ∝ P(C) · Π P(xᵢ | C)   →  choose the argmax class

Often unrealistic, yet it works remarkably well (especially for text).

25. Zero-frequency problem & Laplace smoothing GATE extra

If a feature value never appeared with a class in training, its likelihood is 0 — and one 0 zeroes the whole product.

P(word | class) = 0 ⇒ Π = 0  (kills the prediction)

Fix with Laplace (add-one) smoothing: add a small count so no probability is exactly 0.

26. Naive Bayes — pros, cons, uses GATE extra
  • ✓ simple, fast, great for text, needs little data, handles many features, strong baseline
  • ✗ independence assumption unrealistic, zero-frequency issue, probabilities poorly calibrated

Uses: spam filtering, sentiment analysis, document/news classification, language detection.

28. Feed-forward neural network — the idea GATE extra

A feed-forward neural network (FFNN) sends information one way — input → hidden layers → output — with no cycles or loops. Loosely inspired by neurons in the brain.

input layer → hidden layer(s) → output layer   (information never flows back)
29. A neuron = a tiny logistic-style unit GATE extra

Each neuron does the same two steps as logistic regression: a weighted sum, then an activation.

z = w·x + b ;  a = activation(z)   (e.g. sigmoid / ReLU)

So a network is just many logistic-like units stacked in layers.

30. Hidden layers learn patterns GATE extra

Hidden layers build features of increasing abstraction. Classifying a pizza image:

layer 1: edges → layer 2: shapes (circle, toppings) → output: pizza? yes/no

Stacking layers lets the network learn complex, non-linear patterns a single logistic unit can't.

31. FFNN — when & why GATE extra

Output neurons give the final prediction: one sigmoid for binary, a softmax over several for multi-class, or a linear unit for regression.

binary → 1 sigmoid · multi-class → softmax · regression → linear output

Use FFNNs when relationships are complex and there is plenty of data; they generalize logistic regression.

32. MLP — a multi-layer perceptron GATE extra

An MLP is a feed-forward network with an input layer, one or more hidden layers, and an output layer — the workhorse that learns complex non-linear patterns.

a neuron: z = w·x + b → output = activation(z)   (a layer = many such neurons)
33. Activation functions (and why) GATE extra

Activations inject non-linearity — without them, stacking layers collapses into a single linear model.

sigmoid, tanh, ReLU(x)=max(0,x), softmax (multi-class probabilities)

ReLU is the common hidden-layer choice; softmax sits on a multi-class output.

34. Training: forward pass → loss → update GATE extra 
1. forward pass: compute the prediction
2. compute the loss
3. backpropagation: get gradients
4. optimizer updates weights & biases
5. repeat for many epochs
35. Backpropagation GATE extra

Backpropagation uses the calculus chain rule to find how much each weight contributed to the loss, so the optimizer knows which way to nudge it.

optimizers: gradient descent, SGD, Adam

Strengths: learns complex non-linear maps, flexible, foundation of deep learning. Weaknesses: needs lots of data, compute-heavy, overfits, hard to interpret, needs scaled inputs.

36. Regularizing a neural network GATE extra

Networks overfit easily, so they use their own regularizers (beyond L1/L2 on weights):

  • Dropout — randomly switch off some neurons each training step, so the net can't lean on any one neuron ("random neuron sleeping").
  • Early stopping — halt training once validation error starts rising (training error still falling = overfitting begins).
  • Data augmentation — create more training data by transforming it (rotate/flip/crop/brighten images).
  • Batch normalization — normalize layer activations; stabilizes & speeds training, with a mild regularizing effect.
37. Feed-forward vs recurrent GATE extra

An FFNN/MLP flows one way (input → hidden → output) with no cycles. A recurrent network (RNN) has loops that feed a layer's output back in — giving it memory for sequences (text, time series).

feed-forward = no loops (static input) · recurrent = loops (sequential data)
38. Deep learning — many-layer networks GATE extra

Deep learning = neural networks with many layers that learn patterns hierarchically. It shines on images, audio, text, video and other large, complex data.

more layers → more abstract features learned automatically
39. CNN — built for images GATE extra

A Convolutional Neural Network is designed for image-like data; it learns visual patterns with sliding filters instead of fully connecting every pixel.

pixels → edges → shapes → object parts → full object
40. Filters / kernels & feature maps GATE extra

A filter (kernel) is a small matrix (e.g. 3×3) that slides over the image detecting a pattern (edge, corner, texture). The result is a feature map showing where that pattern occurs.

$\begin{bmatrix}1&0&-1\\1&0&-1\\1&0&-1\end{bmatrix}$ → detects vertical edges

In CNNs, filters are learned from data, not hand-designed.

41. Convolution layer GATE extra

The convolution layer applies many filters to the input (or previous feature maps) to extract local patterns.

early layers: edges, corners, textures
deeper layers: parts, objects, complex shapes
42. Pooling layer GATE extra

Pooling downsamples a feature map — less computation, more robustness, less overfitting.

region $\begin{bmatrix}1&3\\2&4\end{bmatrix}$: max pooling → 4 · average pooling → 2.5

Pooling reduces spatial size (it never enlarges it).

43. Stride & padding GATE extra 
stride  = how many pixels the filter jumps each step (stride 2 → smaller output)
padding = extra border pixels:
   valid = no padding · same = output size ≈ input size (preserves edges)
44. Receptive field GATE extra

The receptive field is the region of the input image that influences one output neuron.

early layers: small local patch · deeper layers: larger area of the image
45. Fully connected layer & CNN architecture GATE extra

After convolution + pooling, features are flattened and fed to fully connected layers that combine them for the final prediction.

input → [conv → activation → pool] × N → flatten → fully connected → output
46. CNN — activations, hyperparameters, traps GATE extra

Hidden layers use ReLU; Leaky ReLU allows small negatives to avoid "dead" neurons; softmax sits on a multi-class output.

CNN hyperparameters: #filters, filter size, stride, padding, pooling size, #layers, lr, epochs, dropout
  • CNN filters are fixed by hand? No — learned from data.
  • Pooling enlarges the image? No — it shrinks it.

Uses: image classification, face recognition, medical imaging, object detection, OCR, self-driving vision.

47. Object detection — what + where GATE extra

Detection goes beyond classification — it finds multiple objects and their locations.

classification → "a dog"     localization → "dog in this box"
detection      → "dog here, cat here, car here" (label + box for each)

A bounding box is (xmin, ymin, xmax, ymax); output = class + confidence + box.

48. Detectors: two-stage vs one-stage GATE extra 
two-stage (propose regions, then classify): R-CNN → Fast R-CNN → Faster R-CNN (RPN)
   accurate but slower
one-stage (predict boxes + classes in one pass): YOLO, SSD
   fast, real-time

YOLO splits the image into a grid; each cell predicts boxes + class probabilities in a single look.

49. IoU, anchor boxes & NMS GATE extra 
IoU = area(intersection) / area(union)   → overlap of predicted vs true box; ≥0.5 = correct

Anchor boxes: predefined shapes (tall for person, wide for car) the model adjusts to fit objects.

Non-Max Suppression: keep the highest-confidence box, drop others that overlap it heavily — removes duplicate detections.

50. Face recognition & one-shot learning GATE extra 
verification = "same person?" (yes/no)   recognition = "who is this?" (identity)

Modern systems map a face to an embedding vector (CNN); same person → close vectors, different → far.

One-shot learning recognizes from one photo. A Siamese network (twin shared-weight nets) compares two faces; triplet loss pulls anchor–positive close and pushes anchor–negative far (FaceNet).

51. Neural style transfer GATE extra

Combine the content of one image with the style of another (your photo painted like Van Gogh).

content loss → keep objects/structure
style loss   → match texture/colors, measured with a Gram matrix

The Gram matrix captures correlations between CNN feature maps — i.e. which visual patterns co-occur (the "style").

52. Advanced CNN tricks (ResNet, Inception, 1×1) GATE extra
  • ResNet / skip connections: output = F(x) + x; the shortcut lets gradients flow, so very deep nets train (fights vanishing gradients).
  • Inception: apply 1×1, 3×3, 5×5 & pooling in parallel, then concatenate — features at multiple scales.
  • 1×1 convolution: mixes/reduces channels cheaply and adds non-linearity.
53. RNN — networks for sequences GATE extra

A Recurrent Neural Network handles ordered data (text, speech, time series) by keeping a hidden state (memory):

hₜ = f(hₜ₋₁, xₜ)   — current memory depends on previous memory + current input

The same weights are reused at every time step (so any length works); trained by backpropagation through time (BPTT).

54. Vanishing & exploding gradients GATE extra 
vanishing: gradients → 0 over long sequences → can't learn long-range dependencies (plain RNN's weakness)
exploding: gradients → ∞ → unstable training, loss → NaN

Fixes: gradient clipping (cap the gradient norm) for exploding; gated units (LSTM/GRU) and good initialization (Xavier/He) for vanishing.

55. LSTM & GRU (gated RNNs) GATE extra

LSTM adds a cell state + 3 gates so information survives many steps (eases vanishing gradients):

forget gate (drop old) · input gate (add new) · output gate (expose useful)

GRU is a lighter version with 2 gates (update, reset) — often faster, similar performance.

56. Word representation: one-hot vs embeddings GATE extra 
one-hot: cat=[1,0,0,0], dog=[0,1,0,0] — huge, sparse, NO similarity (cat as unrelated to dog as to king)
embedding: dense vector capturing meaning — similar words are close
famous analogy:  king − man + woman ≈ queen

An embedding matrix is V × d (vocabulary × dimension); a word index looks up its row.

57. Word2Vec (CBOW & skip-gram) GATE extra 
CBOW:      context → target word   ("I love ___ very much" → pizza)
skip-gram: target word → context   (pizza → I, love, very, much) — better for rare words

Negative sampling speeds training by contrasting real (pizza, eat) vs fake (pizza, galaxy) pairs instead of scoring the whole vocabulary. t-SNE visualizes embeddings in 2-D.

58. Language models & perplexity GATE extra

A language model gives the probability of a word sequence and predicts the next word:

P(w₁…wₙ) = ∏ P(wₜ | w₁…wₜ₋₁)   (chain rule)
n-gram: approximate with previous n−1 words (bigram, trigram) — limited long context

Perplexity = how "surprised" the model is; Perplexity = P(w₁…wₙ)^(−1/n) = 2^entropy. Lower is better.

59. Decoding: beam search & BLEU GATE extra

Beam search keeps the top-k partial sequences at each step (k = beam width) instead of only the single best — wider beam = better but slower; length normalization stops it favouring short sentences.

BLEU scores machine translation by n-gram overlap with a reference — but it misses valid paraphrases (different words, same meaning → low BLEU).

60. Attention & self-attention GATE extra

Attention lets the model focus on the most relevant input parts (softmax weights summing to 1) instead of cramming everything into one vector — fixing the encoder-decoder bottleneck on long sequences.

Self-attention: tokens attend to other tokens in the same sequence (resolving "it" → "animal"). The Transformer core:

Attention(Q,K,V) = softmax(QKᵀ / √dₖ) · V
Q = what I seek · K = what each token offers · V = info to take · √dₖ keeps it stable
61. Where cross-entropy comes from (MLE)

Logistic regression's loss isn't arbitrary — it's maximum likelihood. For one point P(y|x)=pʸ(1−p)¹⁻ʸ; over all data the log-likelihood is

ℓ = Σ [ yᵢ log pᵢ + (1−yᵢ) log(1−pᵢ) ]

Maximizing ℓ = minimizing its negative = exactly binary cross-entropy. (Likewise, minimizing squared error in linear regression = MLE under Gaussian noise.)

62. Transfer learning GATE extra

Reuse a model trained on a big task as the starting point for a new, smaller one — its early layers already learned general features (edges, textures, word patterns).

ImageNet CNN → fine-tune for medical images (less data, faster, better)

Freeze early layers, retrain the later ones for your task.

63. Common GATE traps
  • Classification predicts numbers? No — categories (regression predicts numbers).
  • Binary = many classes? No — two; multi-class = more than two.
  • Code nominal categories as 0,1,2? No — one-hot, to avoid fake order.
  • Probability is the label? No — convert via threshold (binary) or argmax (multi-class).
  • Accuracy is always enough? No — imbalanced data needs precision/recall/F1.

6 Main formulas

From score to probability

Sigmoid & linear score:

$$ \sigma(z)=\frac{1}{1+e^{-z}}, \qquad z=\beta_0+\beta_1 x_1+\dots+\beta_p x_p $$

Probability & log-odds (logit):

$$ \hat{p}=\sigma(z),\qquad \log\!\frac{p}{1-p}=z $$

Training & decision

Log-loss (binary cross-entropy) — the cost minimized over the data:

$$ J=-\frac1n\sum_{i=1}^{n}\big[\,y_i\log\hat{p}_i+(1-y_i)\log(1-\hat{p}_i)\,\big] $$

Decision rule — compare the probability to the threshold:

$$ \hat{y}=1 \text{ if } \hat{p}\ge 0.5\ (z\ge 0),\quad \text{else } 0 $$
7 Step-by-step method
  1. Compute the linear score $z=\beta_0+\beta^{\mathsf T}x$.
  2. Apply the sigmoid: $\hat{p}=\sigma(z)$.
  3. Classify: predict 1 if $\hat{p}\ge0.5$ (i.e. $z\ge0$), else 0.
  4. Train by minimizing log-loss (via gradient descent) to find $\beta$.
8 Worked example
✦ Probability from a score

If $z=0$, $\hat{p}=\sigma(0)=\tfrac{1}{1+1}=0.5$ (right on the boundary).

If $z=2$, $\hat{p}=\tfrac{1}{1+e^{-2}}\approx0.88$ → predict class 1.

9 GATE cheat sheet
⚡ High-value facts
  • $\sigma(0)=0.5$; $\sigma(-z)=1-\sigma(z)$; output always in $(0,1)$.
  • Decision boundary is linear in $x$ (where $z=0$) — same shape as linear regression's line.
  • Logistic regression models the log-odds as linear: $\log\frac{p}{1-p}=z$.
  • Trained by minimizing log-loss (cross-entropy), not squared error.
  • It outputs a probability — threshold at 0.5 (or another value) to decide the class.
10 Common traps
✕ Watch out

Trap: "Logistic regression predicts a number (regression)." Reality: it's a classifier outputting a probability.

Trap: "It minimizes squared error." Reality: it minimizes log-loss (cross-entropy).

Trap: "The decision boundary is curved." Reality: in the raw features it's linear ($z=0$).
