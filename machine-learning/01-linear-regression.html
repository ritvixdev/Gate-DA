Topic 1 · GATE weight: Very High Core

Linear Regression

Draw the best straight line through a cloud of points — then use it to predict. It's the simplest model in ML, and the gateway to everything else.

2 Why this topic exists

We often want to predict a number — a price, a score — from some inputs. Linear regression assumes the output is roughly a weighted sum of the inputs and finds the best weights, giving a simple, interpretable predictor and a baseline every ML course starts with.

4 Basic definitions
  • Feature / predictor $x$: an input variable. Target $y$: the number we want to predict.
  • Simple vs multiple: one feature vs several features $x_1,\dots,x_p$.
  • Coefficients (weights) $\beta_j$: how much each feature adds. Intercept $\beta_0$: the baseline output when every $x=0$.
  • Prediction $\hat{y}$: the model's output. Residual: the error $y-\hat{y}$ at one point.
  • Least squares: the fitting rule — pick $\beta$ to make the total squared residual smallest.
  • Cost function (MSE): the average squared error the fit minimizes.
  • $R^2$ (goodness of fit): the fraction of the target's variance the model explains (1 = perfect).
  • Overfitting: matching noise in the training data, so error on new data rises.
📖 Study in Depth — supervised learning, predictive models, linear regression & error metrics
1. What supervised learning is

The model learns from labeled data — examples where the answer is already given:

study hours = 5  → marks = 75
email text       → spam / not spam
house = 1000 sqft → price = ₹50 lakh

After seeing many such examples it learns the pattern, then predicts the answer for a new input. Linear regression (this topic) is the simplest supervised algorithm.

2. Labeled data = features (X) + target (Y) 
X = inputs / features   (height, weight, age, pixels, text …)
Y = output / label / target   (price, spam?, cat/dog …)

Every training row carries both: the question (X) and its correct answer (Y). No labels ⇒ it isn't supervised learning.

3. Teacher–student mental model

"Supervised" = there is a teacher. A teacher shows a child many labelled pictures (cat, dog, car); the child learns, then names a new picture correctly.

training data = teacher's examples · labels = correct answers
model = student · algorithm = the learning process · prediction = the child's answer
4. The goal: learn a function f(X) ≈ Y 
X  →  f (model)  →  Ŷ  (prediction)
goal: f(X) ≈ Y  on new, unseen data
f(study hours = 5) → 75 marks

Training picks the function that best reproduces the known answers and generalizes beyond them.

5. The supervised-learning workflow 
1. collect labeled data
2. split into training + test sets
3. choose an algorithm
4. train the model
5. evaluate on the test set
6. predict on new data

e.g. past house data → linear regression → price model → predict a new house. (Why the train/test split matters — generalization, overfitting — is covered in the Cross-Validation topic.)

6. Regression vs classification
TypeOutputExamples
Regressiona number (continuous)price, marks, temperature, rainfall mm
Classificationa category (discrete)spam/not, cat/dog, pass/fail, rain/no-rain
Regression predicts a quantity · Classification predicts a class

Linear regression is the canonical regression method; logistic regression (later topic) handles classification.

7. Error & the loss function

During training the model predicts, and we compare with the truth:

actual = 80, predicted = 75 → error = 5

A loss function scores the total error; training tries to minimize it.

regression loss: MSE, MAE   ·   classification loss: cross-entropy
8. Training = optimization

Training is an optimization problem: find the parameters that minimize the loss.

linear model: ŷ = w·x + b      (w = weight/slope, b = bias/intercept)
methods: least squares (closed form) · gradient descent (iterative)

So "learning" literally means searching parameter space for the lowest-loss model.

9. The algorithm toolbox

Many algorithms (kNN, trees, neural nets) can do either task depending on the output.

10. Why machine learning at all?

Use ML when rules are hard to hand-write but data is plentiful. You can't easily code "if ears triangular and whiskers… then cat", but a model can learn it from thousands of labelled images.

complex pattern + lots of labelled data → let the model learn the rule
11. Connections — linear algebra & probability

Data is stored as matrices/vectors: X = feature matrix (rows = examples, columns = features), y = target vector. So least squares, projections and matrix multiply all reappear here.

Classifiers often output probabilities: P(spam | email) = 0.92 — logistic regression, Naive Bayes and neural nets all do this.

12. Applications
  • spam & fraud detection, credit scoring, loan approval
  • medical diagnosis, insurance premium prediction, churn prediction
  • house-price prediction, weather forecasting, object recognition
  • speech recognition, sentiment analysis, recommendations

Theme: wherever labelled historical data exists, supervised learning applies.

13. Predictive model — input → prediction

A predictive model takes inputs and produces an output for an unknown/future case — it answers "given this information, what is likely to happen?"

age, BMI, smoker → predicted premium
humidity, pressure, temperature → rain / no rain

Not every predictive model is machine learning — there are three flavours below.

14. Three kinds of predictive model
  • Numerical: a mathematical equation.
  • Rule-based: human-written if–else logic.
  • ML-based: rules learned automatically from data.
numerical → equation · rule-based → human logic · ML-based → data-learned logic
15. Numerical model (the equation view)

A numerical model is just a formula:

house price = 5000·area + 200000     (area=1000 → ₹52,00,000)
marks       = 20 + 10·hours          (hours=5 → 70)

Linear regression is exactly this, with the numbers learned:

ŷ = w₁x₁ + w₂x₂ + … + wₙxₙ + b
16. Rule-based model & why it doesn't scale 
if smoker: premium += 10000
if age > 50: premium += 15000
if BMI > 30: premium += 8000

Clear for simple problems, but with many variables (age, BMI, city, history, claims, …) the combinations explode, rules conflict, and updating becomes a nightmare.

more variables → more rules → unmaintainable
17. Rule-based vs ML-based
Rule-basedML-based
Howhuman writes ruleslearns from data
Many variablesbecomes hardhandles well

Updatingedit rules by handretrain on new data

ML can even learn interactions — e.g. smoking risk that grows with age — which fixed rules miss.

18. Simple linear regression — ŷ = β₀ + β₁x

Assume an approximately straight-line relation between one feature and the target:

ŷ = β₀ + β₁x        (β₀ intercept, β₁ slope)   also written ŷ = b + wx
marks = 20 + 10·hours → hours=6 → 80
19. What slope & intercept mean 
slope β₁     = change in ŷ per +1 in x   (rate of change)
intercept β₀ = predicted ŷ when x = 0    (starting value)
marks = 20 + 10·hours: each extra hour → +10 marks; 0 hours → 20 marks
20. Multiple linear regression

Several features, one weight each:

ŷ = β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ
premium = 2000 + 300·age + 500·BMI + 10000·smoker
age=40, BMI=28, smoker=1 → 2000+12000+14000+10000 = ₹38,000

Categorical inputs are encoded first (smoker: yes=1, no=0).

21. Parameters, actual vs predicted, residual 
parameters learned in training: β₀, β₁, …, βₙ  (weights + bias)
actual y · predicted ŷ · residual  e = y − ŷ
y=80, ŷ=75 → e = 5  (model predicted 5 low); e=0 → perfect for that point
22. Scatter plot — reading the relationship

Plot each example as a point (x vs y). The shape reveals the relationship:

upward cloud   → positive linear
downward cloud → negative linear
round blob     → weak / no linear relationship

It also exposes non-linear patterns and outliers before you fit anything.

23. Many lines — which is best?

Infinitely many lines fit a scatter; the best one has the smallest overall error.

data (1,30)(2,40)(3,50)
A: ŷ=20+10x → 30,40,50  errors 0,0,0
B: ŷ=25+5x  → 30,35,40  errors 0,5,10  →  A is better
24. Why square the errors?

Raw residuals cancel: +5 + (−5) = 0 hides two real mistakes. Squaring fixes both problems:

(+5)² = 25 , (−5)² = 25 → both count; large errors are punished more

So regression minimizes the sum of squared residuals, not the raw sum.

25. RSS — residual sum of squares 
RSS = Σ (yᵢ − ŷᵢ)²     (total squared prediction error; lower = better fit)
y=10,20,30 ; ŷ=12,18,33 → errors −2,2,−3 → squares 4,4,9 → RSS = 17

Least squares (next topic) chooses the parameters that minimize RSS.

26. MSE — mean squared error 
MSE = RSS / n = (1/n) Σ (yᵢ − ŷᵢ)²
RSS=17, n=3 → MSE ≈ 5.67   (units = target², harder to read)
27. RMSE — root mean squared error 
RMSE = √MSE
MSE=5.67 → RMSE ≈ 2.38  → "typical error ≈ 2.38 units"

RMSE is in the same unit as the target (₹, marks…), so it's the readable one. MSE is in squared units.

28. Normalized RMSE

RMSE depends on scale, so normalize it to compare across datasets:

NRMSE = RMSE / (y_max − y_min)      (or RMSE / mean(y))
RMSE=5, range 0–100 → NRMSE = 0.05 = 5%
29. TSS & R² — variance explained 
TSS = Σ (yᵢ − ȳ)²      (total variation around the mean)
R²  = 1 − RSS/TSS      (fraction of variance the model explains)
RSS=20, TSS=100 → R² = 1 − 0.2 = 0.8  → explains 80%
  • R² = 1 perfect · R² = 0 no better than predicting ȳ · R² < 0 worse than the mean
30. Common GATE traps
  • Supervised learning needs no labels? No — labels are required.
  • Classification predicts numbers? No — it predicts categories; regression predicts numbers.
  • High training accuracy = good model? No — could be overfitting; check test performance.
  • Test data is used to train? No — train on the training set, evaluate on the test set.
  • More complex model is always better? No — it may overfit.

6 Main formulas

The model

Prediction — a weighted sum of the features:

$$ \textbf{Simple: } \hat{y}=\beta_0+\beta_1 x \qquad \textbf{Multiple: } \hat{y}=\beta_0+\beta_1 x_1+\dots+\beta_p x_p $$

Least-squares slope & intercept (simple regression):

$$ \beta_1=\frac{\sum (x_i-\bar{x})(y_i-\bar{y})}{\sum (x_i-\bar{x})^2}, \qquad \beta_0=\bar{y}-\beta_1\bar{x} $$

Error & evaluation

Errors — squared error, averaged and square-rooted:

$$ \text{RSS}=\sum (y_i-\hat{y}_i)^2,\quad \text{MSE}=\tfrac{\text{RSS}}{n},\quad \text{RMSE}=\sqrt{\text{MSE}} $$

$R^2$ (coefficient of determination) — the share of variance explained:

$$ \text{TSS}=\sum (y_i-\bar{y})^2,\qquad R^2=1-\frac{\text{RSS}}{\text{TSS}} $$

RMSE is in the target's own units; $R^2\le 1$ ($1$ perfect, $0$ = as good as the mean, negative = worse).

7 Step-by-step method
  1. Compute the means $\bar{x}$ and $\bar{y}$.
  2. Slope $\beta_1=\dfrac{\sum(x_i-\bar{x})(y_i-\bar{y})}{\sum(x_i-\bar{x})^2}$.
  3. Intercept $\beta_0=\bar{y}-\beta_1\bar{x}$.
  4. Predict new points with $\hat{y}=\beta_0+\beta_1 x$.
8 Worked example
✦ Fit a line through (1,2), (2,4), (3,5), (4,4), (5,5)

$\bar{x}=3,\ \bar{y}=4$.   $\sum(x-\bar{x})(y-\bar{y})=(-2)(-2)+(-1)(0)+0+1\cdot0+2\cdot1=6$.

$\sum(x-\bar{x})^2=4+1+0+1+4=10$. So $\beta_1=6/10=0.6$, $\beta_0=4-0.6(3)=2.2$.

Line: $\hat{y}=2.2+0.6x$. Predict $x=6$: $\hat{y}=5.8$.

9 GATE cheat sheet
⚡ High-value facts
  • The least-squares line always passes through $(\bar{x},\bar{y})$.
  • Slope $\beta_1=\operatorname{Cov}(x,y)/\operatorname{Var}(x)=r\cdot s_y/s_x$.
  • $R^2$ = fraction of variance explained (1 = perfect fit, 0 = no better than the mean).
  • Multiple regression in matrix form: $\hat{y}=X\beta$; "linear" means linear in the coefficients.
  • Adding features never increases training error (but can overfit).
10 Common traps
✕ Watch out

Trap: "Correlation implies causation." Reality: a good fit doesn't prove $x$ causes $y$.

Trap: "Predict far outside the data range." Reality: extrapolation is unreliable.

Trap: "Linear means a straight feature relationship." Reality: it means linear in $\beta$; you can use $x^2$ as a feature and still call it linear regression.

11 Connections with other topics
  • Least-squares: the method that finds these coefficients (next topic).
  • Optimization: fitting = minimizing the sum of squared residuals.
  • Linear algebra: multiple regression solves the normal equations $X^{\mathsf T}X\beta=X^{\mathsf T}y$.
  • Covariance: the slope is $\operatorname{Cov}(x,y)/\operatorname{Var}(x)$.
12 Practice problems

Q1. The least-squares line always passes through which point?

$(\bar{x},\bar{y})$ — the means.

Q2. $\operatorname{Cov}(x,y)=6$, $\operatorname{Var}(x)=10$. Slope $\beta_1$?

$\beta_1=6/10=0.6$.

Q3. If $\bar{x}=3,\bar{y}=4,\beta_1=0.6$, find the intercept.

$\beta_0=\bar{y}-\beta_1\bar{x}=4-1.8=2.2$.

Q4. $y=10,20,30$; $\hat{y}=12,18,33$. Find RSS, then MSE and RMSE.

errors $-2,2,-3$ → RSS$=4+4+9=17$; MSE$=17/3\approx5.67$; RMSE$\approx2.38$.

Q5. RSS$=20$, TSS$=100$. Find $R^2$ and interpret.

$R^2=1-20/100=0.8$ → the model explains 80% of the target's variance.

Quick MCQ

In linear regression, "linear" means the model is linear in the…
You can transform features (e.g. $x^2$) and it's still linear regression.
✅ Correct — linear in the $\beta$'s; that's what makes least-squares work.
The target is what we predict, not the linearity condition.
Residuals are the errors, not the linearity.
