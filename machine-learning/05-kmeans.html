k-Means Clustering

No labels? Group the data anyway. k-Means finds $k$ centers and repeatedly snaps points to their nearest center — the classic unsupervised algorithm.

2 Why this topic exists

Often there are no labels — just data. Clustering discovers natural groups. k-Means is the simplest and most-tested: pick $k$ cluster centers, assign each point to the nearest, move the centers to the group averages, and repeat until stable.

4 Basic definitions
  • Cluster: a group of similar points. Centroid: the mean (center) of a cluster.
  • k: the number of clusters you choose in advance.
  • Inertia (WCSS): total squared distance of points to their centroid — k-Means minimizes it.
  • Unsupervised: no labels are used; structure is found from the data alone.
📖 Study in Depth — unsupervised learning, k-means & hierarchical clustering from zero
1. Unsupervised learning — no labels

Unlike supervised learning, there is no target Y — only inputs X. The model finds structure on its own.

supervised:   X + labels → learn f(X)≈Y
unsupervised: X only      → discover patterns/groups
2. What unsupervised learning does
  • Clustering — group similar points (k-means, hierarchical)
  • Association rules — "people who buy X also buy Y"
  • Collaborative filtering — recommendations
  • Dimensionality reduction — compress features (PCA)
3. Clustering — grouping similar points

Clustering puts similar points together and separates dissimilar ones — with no labels to guide it.

good clusters: tight inside, far apart from each other

Uses: customer segmentation, image/document grouping, anomaly detection.

4. K-means — the idea

Partition data into K clusters, each represented by a centroid (its mean point). A point belongs to the nearest centroid.

K = number of clusters (you choose it)
centroid = average of the points currently in that cluster
5. WCSS / inertia — the objective

K-means minimizes the within-cluster sum of squares (also called inertia):

WCSS = Σ_clusters Σ_points ‖ point − centroid ‖²

Low WCSS = tight clusters. Adding more clusters always lowers WCSS (that's why we need the elbow method, below).

6. The k-means algorithm (assign ↔ update) 
1. pick K; initialise K centroids
2. ASSIGN: put each point in its nearest centroid's cluster
3. UPDATE: move each centroid to the mean of its points
4. repeat 2–3 until centroids stop moving (converged)

Each round can only lower WCSS, so it converges — though possibly to a local optimum.

7. Choosing K — the elbow method

Plot WCSS vs K. It always falls, but the drop suddenly flattens — the elbow marks a good K.

K:    1    2    3    4    5
WCSS: 900  400  170  150  140   → elbow at K=3

Before the elbow each extra cluster helps a lot; after it, barely.

8. Feature scaling matters

K-means uses distance, so a large-range feature dominates — scale features first (standardize / min-max).

income 0–100000 vs age 0–100 → income hijacks the clustering unless scaled
9. Interpreting clusters (worked intuition)

After clustering students on performance, you read each cluster's centroid to name it:

high marks + high effort      → "high achievers"
high effort + low marks       → "remedial"
low effort + decent marks     → "careless" / "conservative"

Cluster numbers are arbitrary — you relabel them by what the centroid means.

10. K-means limitations
  • must choose K in advance
  • sensitive to initial centroids (run several times / k-means++)
  • sensitive to scaling & outliers
  • assumes roughly round, similar-size clusters
11. Hierarchical clustering — two directions

Builds a tree of clusters; no need to fix K upfront.

Agglomerative (bottom-up): start with each point alone, merge the closest pairs
Divisive (top-down):       start with one big cluster, split repeatedly

Agglomerative is the common one.

12. Dendrogram — the cluster tree

A dendrogram draws the merge history; the height of a join = distance at which clusters merged.

cut the tree at a chosen height → that many clusters

So you can pick the number of clusters after seeing the structure.

13. Linkage — how to measure cluster distance 
single linkage   = distance between the two closest points (can chain)
complete linkage = distance between the two farthest points (compact clusters)
average linkage  = average pairwise distance

Linkage choice changes the shape of the clusters formed.

14. Distance metrics 
Euclidean   = √Σ(aᵢ−bᵢ)²        straight-line distance
Manhattan   = Σ|aᵢ−bᵢ|           grid / city-block distance
Cosine      = 1 − cosθ            angle between vectors (good for text)

Cosine ignores magnitude — ideal when only direction matters (documents).

15. Hierarchical vs k-means

Applications: market segmentation, document/image clustering, gene-expression, anomaly detection.

16. Evaluating clusters — the silhouette score

WCSS/inertia only measures tightness; the silhouette score also rewards separation.

for each point: compare its avg distance to its own cluster (a) vs the nearest other cluster (b)
silhouette = (b − a) / max(a, b) ,  range −1 … +1
  • near +1 → well inside its cluster, far from others (good)
  • near 0 → on a boundary between clusters
  • negative → probably in the wrong cluster

Average it over all points to compare clusterings (and even to help pick K).

17. Gaussian Mixture Model (soft clustering)

A GMM models data as a blend of K Gaussians — giving each point a probability of belonging to each cluster (soft assignment), unlike k-means' hard assignment.

p(x) = Σₖ πₖ · N(x | μₖ, Σₖ)     πₖ = mixture weight, μₖ mean, Σₖ covariance

Fit by the EM algorithm; handles elliptical/overlapping clusters k-means can't. Used for soft clustering, density estimation, anomaly detection.

18. Common GATE traps
  • k-means needs labels? No — it's unsupervised.
  • More clusters = better (lower WCSS)? Misleading — WCSS always drops; use the elbow.
  • Skip feature scaling? No — distance is scale-sensitive.
  • Hierarchical needs K upfront? No — cut the dendrogram afterwards.
  • k-means always finds the global best? No — can land in a local optimum (depends on init).

20. Memory box & mental model
  • Unsupervised = no labels; clustering groups similar points.
  • k-means: assign to nearest centroid ↔ recompute centroid; minimizes WCSS; pick K via elbow; scale features.
  • Hierarchical: agglomerative/divisive → dendrogram; single/complete/average linkage.
  • Distances: Euclidean, Manhattan, cosine.
k-means = K moving magnets pulling nearby points; centroids settle where WCSS is lowest.
5 Symbols & their meaning
SymbolRead asMeaning
$k$"k"number of clusters

$C_j$"C j"the set of points in cluster $j$
WCSS"inertia"within-cluster sum of squares
6 Main formula
$$ \min \ \sum_{j=1}^{k}\sum_{x\in C_j}\lVert x-\mu_j\rVert^2 \qquad \mu_j=\frac{1}{|C_j|}\sum_{x\in C_j} x $$

Assign each point to the nearest $\mu_j$; recompute each $\mu_j$ as its cluster's mean; repeat.

7 Step-by-step method (Lloyd's algorithm)
  1. Choose $k$ and initialize $k$ centroids (often random points).
  2. Assign: put each point in the cluster of its nearest centroid.
  3. Update: move each centroid to the mean of its assigned points.
  4. Repeat assign/update until assignments stop changing (converged).
8 Worked example
✦ One update

A cluster holds points $(2,2),(4,2),(3,5)$. New centroid = mean = $\big(\tfrac{2+4+3}{3},\tfrac{2+2+5}{3}\big)=(3,3)$.

Each point then re-checks which centroid is nearest, and the loop continues.

9 GATE cheat sheet
⚡ High-value facts
  • Two steps per iteration: assign (nearest centroid) then update (cluster mean).
  • It minimizes the within-cluster sum of squares (inertia); inertia never increases.
  • Converges to a local optimum — the result depends on the initialization.
  • You must choose $k$ in advance (e.g. via the "elbow" of inertia vs $k$).
  • Unsupervised: no labels involved (unlike kNN, which uses labeled neighbors).
10 Common traps
✕ Watch out

Trap: "k-Means finds the global optimum." Reality: only a local one; restart with different seeds.

Trap: "k-Means and kNN are the same." Reality: k-Means is unsupervised clustering; kNN is supervised classification.

Trap: "$k$ is learned automatically." Reality: you set $k$ yourself.

11 Connections with other topics
  • kNN: both use distance, but k-Means clusters (unsupervised), kNN classifies (supervised).
  • Optimization: the assign/update steps decrease the inertia objective.
  • Descriptive stats: a centroid is just the mean of its cluster.
