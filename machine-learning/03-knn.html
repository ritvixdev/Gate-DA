k-Nearest Neighbors

The laziest learner: to classify a new point, just look at its k closest known examples and take a majority vote. No training, no formula — only distance.

2 Why this topic exists

Sometimes "similar things behave similarly" is all you need. kNN classifies a new point by its nearest neighbors in the data — simple, intuitive, and a great baseline that needs no training phase (it just stores the data).

4 Basic definitions
  • kNN: classify a query by the majority label of its $k$ nearest training points (or average their values for regression).
  • $k$: how many neighbours vote — a hyperparameter you choose (an odd $k$ avoids ties).
  • Distance metric: how "nearness" is measured — usually Euclidean, sometimes Manhattan.
  • Feature scaling: standardize features first, so one large-range feature doesn't dominate the distance.
  • Lazy learner: no real training — all the work happens at prediction time.
  • Decision boundary: the dividing line between predicted classes; small $k$ makes it jagged, large $k$ smooth.
  • Bias–variance with $k$: small $k$ = low bias / high variance (overfit); large $k$ = high bias / low variance (underfit).
  • Curse of dimensionality: in many dimensions all points look about equally far, so kNN weakens.
📖 Study in Depth — classic classifiers: kNN, SVM & decision trees from zero
1. What kNN is

k-Nearest Neighbours classifies a new point by looking at the k closest training points and taking their majority class.

"Tell me who your nearest neighbours are, and I'll guess your class."

A new player's skill? Look at the nearby players (kills, accuracy, playtime) — if most are experts, call them an expert.

2. Distance — how 'near' is measured

kNN ranks neighbours by distance, usually Euclidean:

2-D:  d = √[(x₂−x₁)² + (y₂−y₁)²]
n-D:  d = √[ Σ (featureᵢ difference)² ]

Closest = smallest distance in feature space.

3. The kNN algorithm 
1. choose k
2. compute distance from the new point to every training point
3. take the k smallest (nearest) points
4. count their class labels
5. assign the majority class
k=3, nearest labels = spam, spam, not-spam → predict spam
4. Worked example 
new point's 5 nearest labels (k=5): cat, cat, dog, cat, dog
votes: cat 3, dog 2 → predict cat

For regression, kNN averages the neighbours' values instead of voting.

5. Choosing k (a hyperparameter)

Hyperparameter = a value you set before training, not learned from data.

small k → sensitive to noise, can overfit (jagged boundary)
large k → smoother boundary, can underfit (ignores local detail)

Use an odd k for binary problems to avoid tie votes; pick the best k by validation.

6. Effect of k on the boundary 
k = 1   → very flexible, fits every point (high variance, overfit)
k = n   → predicts the global majority for everything (underfit)

The sweet spot lies in between and is found by cross-validation.

7. Lazy learning — no real 'training'

kNN is a lazy learner: it learns no equation. "Training" just means storing the labelled examples; all the work happens at prediction time (computing distances).

train: memorize data · predict: find neighbours & vote

This makes prediction slow on large datasets.

8. Feature scaling is essential

Because kNN uses distance, features on bigger numeric ranges dominate it:

income 0–100000 vs age 0–100 → income swamps the distance

So scale features (standardize / min-max) before kNN, or the result is meaningless.

9. Pros & cons
  • ✓ simple, intuitive, no training, no strong assumptions, does classification and regression
  • ✗ slow at prediction (large data), needs feature scaling, hurt by irrelevant features & high dimensions, k must be tuned
10. SVM — the maximum-margin classifier GATE extra

A Support Vector Machine separates classes with the boundary that has the widest margin — a wider gap usually generalizes better.

SVM = maximum-margin classifier
11. Hyperplane & margin GATE extra 
hyperplane = the decision boundary (2-D: a line, 3-D: a plane, n-D: a hyperplane)
margin = distance from the boundary to the nearest points of each class

SVM maximizes that margin.

12. Support vectors GATE extra

Support vectors are the points closest to the boundary — they alone fix the hyperplane.

move a non-support-vector → boundary barely changes
move a support vector → boundary moves
13. Hard vs soft margin (slack) GATE extra 
hard margin: perfect separation, no mistakes allowed (fails on noisy data)
soft margin: allow some violations via slack ξᵢ ≥ 0

ξᵢ=0 correct & outside margin; ξᵢ>0 inside margin or misclassified. Soft margin trades a wider margin against a few errors.

14. The C parameter GATE extra 
large C → heavy penalty for mistakes → narrow margin → can overfit (strict teacher)
small C → tolerate mistakes → wider margin → more regularization → can underfit (lenient)

Trap: high C means less regularization, not more.

15. The kernel trick GATE extra

When classes aren't linearly separable (e.g. one class inside a circle), the kernel trick implicitly maps data to a higher-dimensional space where a hyperplane can split them — without computing the mapping explicitly.

non-linear boundary in original space = linear boundary in the mapped space
16. Common kernels GATE extra 
linear:      K(x,z) = xᵀz                      (≈ linearly separable)
polynomial:  K(x,z) = (xᵀz + c)^d              (curved)
RBF/Gaussian:K(x,z) = exp(−γ‖x−z‖²)            (very flexible, popular)
sigmoid:     K(x,z) = tanh(α xᵀz + c)

RBF γ: high γ → local, complex boundary (overfit); low γ → smooth (underfit).

17. SVM — pros, cons & scaling GATE extra
  • ✓ strong in high dimensions, kernels handle non-linearity, max-margin generalizes well
  • ✗ slow on huge data, kernel/parameter choice matters, less interpretable, no natural probabilities

Like kNN, SVM is distance-based ⇒ scale features first.

18. Decision tree — questions from root to leaf GATE extra

A decision tree asks yes/no questions, splitting the data until it reaches a class.

root node = first split · internal node = a test · leaf = final prediction
"Salary>50k?" → "Age>30?" → "Credit>700?" → approve / reject
19. Splits & purity GATE extra

A split divides data by a condition (salary > 50000, education = graduate). Good splits make child nodes purer — mostly one class.

Prediction just follows the path from root to a leaf, so trees are very interpretable.

20. Decision trees overfit easily GATE extra

A deep tree can memorize the training data with hyper-specific rules:

if age=37 and salary=51230 and city=X → ...   (won't generalize)
symptom: train accuracy high, test accuracy low
21. Taming a decision tree GATE extra
  • limit max depth · require min samples per leaf · prune
  • use ensembles (Random Forest) · validate with cross-validation
deep tree → high variance · very shallow tree → high bias
22. Entropy — measuring a node's uncertainty GATE extra

A split should make child nodes purer. Entropy measures impurity/uncertainty:

Entropy(S) = − Σ pᵢ log₂(pᵢ)
pure node (10 spam, 0 not) → 0     (no uncertainty)
50/50 node (5,5)               → 1     (max for binary)
23. Information gain GATE extra

A split's quality = how much entropy it removes. First the children's weighted entropy, then the drop:

Weighted Entropy = Σ (child size / total) · Entropy(child)
Information Gain  = Entropy(parent) − Weighted Entropy(children)   (higher = better)
parent (5,5)→1; split into pure (5,0) & (0,5)→0 ⇒ IG = 1 − 0 = 1 (perfect split)

ID3/C4.5 trees pick the feature with the largest information gain.

24. Gini impurity GATE extra

An alternative impurity measure (used by CART) — the chance of mislabelling a random point by the node's class mix:

Gini = 1 − Σ pᵢ²
pure (1,0) → 1 − 1 − 0 = 0
50/50 (0.5,0.5) → 1 − 0.25 − 0.25 = 0.5   (max for binary)
25. Gini gain — worked GATE extra 
parent (6 pos, 4 neg): Gini = 1 − 0.6² − 0.4² = 0.48
left (5,1): 1 − (5/6)² − (1/6)² = 0.278   right (1,3): 1 − (1/4)² − (3/4)² = 0.375
weighted = 0.6(0.278) + 0.4(0.375) = 0.3168
Gini gain = 0.48 − 0.3168 = 0.1632

The tree keeps the split with the largest Gini gain (= lowest weighted Gini).

27. Tree algorithms & regression trees GATE extra
  • ID3 — information gain (classification)
  • C4.5 — gain ratio, handles continuous features
  • CART — Gini (classification) or MSE (regression); binary splits
classification tree → leaf gives a class
regression tree     → leaf gives a number (mean of its training targets)
28. Pre- vs post-pruning GATE extra

Pruning trims the tree to fight overfitting:

pre-pruning: stop early (max_depth, min_samples_leaf, min_samples_split)
post-pruning: grow fully, then cut weak branches

The result generalizes better than a fully-grown tree.

29. SVM optimization (margin, soft margin, dual) GATE extra

Maximizing the margin is equivalent to a clean minimization (the margin is 1/‖w‖):

hard: minimize ½‖w‖²   s.t. yᵢ(wᵀxᵢ + b) ≥ 1
soft: minimize ½‖w‖² + C Σξᵢ   s.t. yᵢ(wᵀxᵢ + b) ≥ 1 − ξᵢ , ξᵢ ≥ 0

Solving via its dual (Lagrange multipliers) depends only on dot products xᵢᵀxⱼ — which is what lets the kernel trick slot in. Only support vectors get non-zero multipliers.

30. SVM & decision tree — traps, practice, memory GATE extra
  • SVM minimizes the margin? No — maximizes it.
  • All points are support vectors? No — only the closest ones.
  • Decision trees never overfit? No — deep ones overfit badly.

Q1. One class sits inside a ring around the other — which SVM?

non-linear: RBF (or polynomial) kernel

Q2. What decides an SVM hyperplane's position?

the support vectors
Memory: SVM = widest-margin boundary set by support vectors; kernels bend it. Tree = if-else path; prune to fight overfit.

  • kNN learns a parameter equation? No — it stores data and votes (lazy).
  • Bigger k is always better? No — too large underfits.
  • Scaling doesn't matter? No — distance is scale-sensitive; always scale.

6 Main formula

Euclidean distance — straight-line distance between two points:

$$ d(a,b)=\sqrt{\sum_{i}(a_i-b_i)^2} $$

Manhattan distance — sum of absolute differences (grid distance):

$$ d(a,b)=\sum_i |a_i-b_i| $$

Classification — predict the majority class among the $k$ nearest points:

$$ \hat{y}_q=\operatorname*{mode}\{\,y_i : x_i \in k\text{ nearest to } x_q\,\} $$

Regression — average the neighbours' values instead:

$$ \hat{y}_q=\frac{1}{k}\sum_{i\in N_k(x_q)} y_i $$
7 Step-by-step method
  1. Pick $k$ and a distance metric.
  2. Compute the distance from the query to every training point.
  3. Take the $k$ smallest distances (the nearest neighbors).
  4. Classification: majority vote. Regression: average their target values.
8 Worked example
✦ k=3 vote

A query's 3 nearest neighbors have labels {A, A, B}. Majority is A → predict class A.

Use odd $k$ for two classes to avoid ties.

9 GATE cheat sheet
⚡ High-value facts
  • No training cost; prediction is $O(n)$ per query (must scan all points).
  • Small $k$ → low bias, high variance (overfits noise); large $k$ → smoother, higher bias.
  • Use odd $k$ for binary classification to avoid ties.
  • Feature scaling matters — distance is dominated by large-range features.
  • kNN can do regression too (average the neighbors' values).
10 Common traps
✕ Watch out

Trap: "kNN has a training phase." Reality: it's a lazy learner — it just stores the data.

Trap: "Bigger $k$ is always better." Reality: too large blurs class boundaries (underfits).

Trap: "No need to scale features." Reality: unscaled features distort distances — always normalize.

11 Connections with other topics

  • k-Means: different idea — same "distance to centers" intuition, but unsupervised.
