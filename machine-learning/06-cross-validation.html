Cross-Validation

How good is your model on data it hasn't seen? Cross-validation estimates that honestly — and stops you from fooling yourself with overfitting.

2 Why this topic exists

A model can ace the data it trained on yet fail on new data (overfitting). Cross-validation reliably estimates real-world performance and helps choose hyperparameters (like $k$ in kNN) — without peeking at the final test set.

3 Game / visual imagination
🎮 Imagine it

Studying for an exam: practicing on the answer key (training) isn't a fair test. Hold out some questions you never studied (validation) to see if you really learned. k-fold CV rotates which chunk is held out, so every example gets a turn as the "exam" — a fairer average score.

4 Basic definitions
  • Train / validation / test: fit on train, tune on validation, judge once on test.
  • Train/test split: the simplest version — fit on one part, evaluate on a held-out part.
  • k-fold CV: split into $k$ folds; train on $k-1$, validate on the rest; rotate; average the scores.
  • LOOCV: leave-one-out — $k$ equals the number of data points.
  • Stratified CV: keep each fold's class proportions the same as the full data.
  • Overfitting: great on training, poor on new data. Underfitting: poor on both (too simple).
  • Bias–variance trade-off: error splits into bias (too simple) and variance (too sensitive).
  • Hyperparameter: a setting you choose (like $k$ or $\lambda$); CV helps pick it without touching the test set.
📖 Study in Depth — generalization, overfitting & cross-validation from zero
1. Why split the data?

If we judge a model on the same data it trained on, it can simply memorize and look perfect. So we split:

training set → fit the model
test set     → judge it on data it has never seen

The test set is the honest exam; the training set is the homework.

2. Generalization — the real goal

Generalization = performing well on new, unseen data, not just the training data.

seen 1000 house prices → should price a brand-new house well

A good model learns the underlying pattern, not the specific rows.

3. Overfitting — memorizing noise

The model fits the training data too closely, capturing noise as if it were signal:

training accuracy HIGH · test accuracy LOW

Like a student who memorized last year's exact questions but can't solve new ones. Causes: model too complex, data too small, noise high, trained too long.

4. Underfitting — too simple

The model is too weak to capture the pattern:

training accuracy LOW · test accuracy LOW

Like a student who never understood the topic. Fix: a richer model / more features.

5. Good fit & the bias–variance balance

A good model sits between the two extremes — learns the signal, ignores the noise, and generalizes.

underfit  → high bias  (too simple)
overfit   → high variance (too sensitive to the training sample)
good fit  → balanced bias & variance

Total error trades off: lowering bias often raises variance, and vice-versa.

6. From one split to k-fold cross-validation

A single train/test split is noisy — the score depends on which rows landed in the test set. k-fold CV fixes this:

split data into k folds
for each fold: train on the other k−1, validate on this fold
average the k scores

Every example gets one turn as validation, so the estimate is fairer and more stable.

7. LOOCV & choosing k 
LOOCV = leave-one-out: k = n (each point is its own validation set)
  • large k → less bias, more computation (LOOCV is the extreme)
  • small k (e.g. 5 or 10) → cheaper, slightly more bias — the common choice

CV is also how we tune hyperparameters (like k in kNN) without touching the final test set.

8. Real-world caveats — bias & drift
  • Biased data → biased model (e.g. historically unfair loan data).
  • Distribution shift → data changes over time (fraud tactics, customer behavior), so accuracy drops and models need retraining.
garbage / biased data → garbage / biased model
9. Bias & variance, precisely 
BIAS     = error from too-simple assumptions  → underfitting (train & test both poor)
VARIANCE = sensitivity to the training sample → overfitting (train good, test poor)

Total error blends both; complexity ↑ lowers bias but raises variance, and vice-versa.

10. The dartboard analogy 
low bias,  low variance  → tight cluster on the bullseye   (best)
high bias, low variance  → tight cluster, but off-centre    (consistently wrong)
low bias,  high variance → around centre but scattered      (unstable)
high bias, high variance → scattered AND off-centre         (worst)
11. Controlling bias & variance 
too much BIAS (underfit): richer model · more features · train longer · less regularization
too much VARIANCE (overfit): regularize · more data · simpler model · prune trees · ensembles · CV

Aim for the complexity where test/validation error is minimum.

12. Cross-validation — stratified, time-series, leakage
  • Stratified k-fold: keep each class's proportion in every fold — use for (imbalanced) classification.
  • Time series: never shuffle — validate chronologically (train on past, test on future).
  • Data leakage: fit scalers/encoders inside each fold, not before splitting, or scores are optimistic.
13. Parameters vs hyperparameters 
parameter      = learned FROM data        (weights w, b; tree split values)
hyperparameter = chosen BEFORE training   (you set it)
examples: k in kNN · max_depth · C & γ in SVM · λ in ridge/lasso · learning rate, #layers in NN
14. Hyperparameter tuning
  • Grid search — try every combination on a grid (systematic, expensive).
  • Random search — sample combinations (often finds good values faster).
  • Bayesian optimization — use past results to choose the next try (good when training is costly).

Always tune with a validation set / cross-validation — never on the test set (which is for final evaluation only).

15. The confusion matrix 
                Predicted +   Predicted −
Actual +           TP            FN
Actual −           FP            TN

TP/TN = correct; FP = false alarm; FN = missed positive. Every classification metric is built from these four counts.

16. Precision — trust of a positive call 
Precision = TP / (TP + FP)   ("when it says positive, how often is it right?")

High precision = few false positives. Matters when a false alarm is costly — e.g. a real email wrongly flagged as spam.

17. Recall — coverage of actual positives 
Recall = TP / (TP + FN)   ("of all real positives, how many did we catch?")

Also called sensitivity / true-positive rate. High recall = few false negatives. Critical when a miss is dangerous — e.g. a disease left undetected.

18. Precision vs recall (the trade-off) 
Precision ↓ false positives · Recall ↓ false negatives

Lowering the decision threshold catches more positives (↑recall) but adds false alarms (↓precision). You pick the balance the application needs.

19. F1-score (and why harmonic mean) 
F1 = 2 · Precision · Recall / (Precision + Recall)

The harmonic mean punishes imbalance — it's only high when both are high.

P=1.0, R=0.1 → arithmetic mean 0.55 but F1 = 2(0.1)/1.1 ≈ 0.18
20. Worked example & when to use F1 
TP=40, FP=10, FN=20, TN=30
Accuracy  = 70/100 = 0.70
Precision = 40/50  = 0.80
Recall    = 40/60  ≈ 0.667
F1        = 2(0.8·0.667)/(0.8+0.667) ≈ 0.727

Use F1 (over accuracy) for imbalanced data where both error types matter — fraud, medical, defect, rare-disease, spam.

21. Traps, practice & memory box
  • High training accuracy ⇒ good model? No — could be overfitting; check validation/test.
  • Tune hyperparameters on the test set? No — use CV / a validation set; keep the test set untouched.
  • Overfit = high bias? No — overfit = high variance; underfit = high bias.

Overfitting (memorized training data, poor generalization)

6 Main idea

k-fold CV score — the average validation score across folds:

$$ \text{CV}=\frac{1}{k}\sum_{i=1}^{k}\text{score}_i \quad(\text{fold } i \text{ held out as validation}) $$

Train on the other $k-1$ folds each time; averaging gives an honest estimate of new-data performance.

Bias–variance decomposition — expected error splits into three parts:

$$ \mathbb{E}\big[(y-\hat{f})^2\big]=(\text{Bias})^2+\text{Variance}+\sigma^2_{\text{noise}} $$

More flexible models cut bias but raise variance; CV finds the sweet spot.

7 Step-by-step method (k-fold)
  1. Shuffle and split the data into $k$ equal folds.
  2. For each fold: train on the other $k-1$, validate on this fold, record the score.
  3. Average the $k$ scores → the cross-validation estimate.
  4. Use it to pick hyperparameters; report final performance on a separate untouched test set.
8 Worked example
✦ 5-fold CV

100 points → 5 folds of 20. Train on 80, validate on 20, five times (each fold held out once).

If validation scores are $0.80,0.82,0.78,0.85,0.80$, the CV estimate is their mean $=0.81$.

9 GATE cheat sheet
⚡ High-value facts
  • $k$-fold trains the model $k$ times; every point is validated exactly once.
  • LOOCV is $k=n$ — nearly unbiased but expensive and higher variance.
  • Large gap (train ≫ validation accuracy) ⇒ overfitting.
  • Both low ⇒ underfitting (model too simple).
  • Never tune on the test set — keep it untouched for the final estimate.
10 Common traps
✕ Watch out

Trap: "Tune hyperparameters on the test set." Reality: that leaks information — use CV / a validation set.

Trap: "High training accuracy = good model." Reality: it may be overfitting; check validation.

Trap: "More folds is always better." Reality: more folds cost more compute and can raise variance (LOOCV).
