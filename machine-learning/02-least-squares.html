Least-Squares

"Best fit" needs a definition of best. Least-squares picks the line that makes the total squared error as small as possible.

2 Why this topic exists

Many lines pass near the data — which is "best"? Least-squares answers it precisely: minimize the sum of squared residuals (vertical gaps). Squaring punishes big misses and gives a clean, solvable formula — the foundation of regression.

4 Basic definitions
  • Residual $e_i=y_i-\hat{y}_i$: the vertical gap between a point and the fitted line.
  • SSE / RSS: $\sum e_i^2$ — the total squared error least-squares minimizes.
  • Design matrix $X$: the table of feature values (one row per point, plus a 1s column for the intercept).
  • Normal equations: the linear system whose solution gives the optimal coefficients.
  • Ordinary Least Squares (OLS): the standard least-squares fit with no penalty.
  • Closed form vs gradient descent: solve the equations directly, or roll downhill to the same minimum.
  • Regularization: add a penalty on big coefficients — Ridge (L2) shrinks them, Lasso (L1) can zero them out.
📖 Study in Depth — least squares, normal equations, projection, ridge & lasso
1. What least squares does

Least squares is the method that picks the regression line/plane — it chooses the parameters that make the squared residuals as small as possible.

residual eᵢ = yᵢ − ŷᵢ ;  objective: minimize Σ eᵢ²

We square (not sum raw errors) so positives and negatives can't cancel and big misses cost more.

2. The objective: minimize RSS 
min over β   RSS = Σ (yᵢ − ŷᵢ)² = Σ (yᵢ − (β₀ + β₁xᵢ))²

Training a linear model literally means solving this minimization for β₀, β₁, … . The minimum is where the gradient w.r.t. each β is zero.

3. Closed form for simple regression

Setting the two derivatives to zero gives a direct formula (no iteration):

β₁ = Σ(xᵢ−x̄)(yᵢ−ȳ) / Σ(xᵢ−x̄)² = Cov(x,y)/Var(x)
β₀ = ȳ − β₁·x̄

So the slope is just covariance over variance, and the line always passes through (x̄, ȳ).

4. Matrix form of regression

Stack all examples into matrices:

y = Xβ + ε        ŷ = Xβ
X = feature matrix (rows = examples, cols = features, plus a 1-column for the intercept)
y = target vector ,  β = parameter vector ,  ε = errors

This is the bridge from linear regression to linear algebra.

5. The normal equations

Minimizing ‖y − Xβ‖² and setting the gradient to zero gives the normal equations:

XᵀX β = Xᵀy     ⇒     β̂ = (XᵀX)⁻¹ Xᵀy

One linear solve gives all coefficients at once — the closed-form OLS solution.

6. MSE gradient in vector form (GD vs closed-form)

Besides the normal equations, you can reach the same β by gradient descent. Write the loss and its gradient in matrix form:

J(w) = (1/n)‖Xw − y‖²        (Xw − y = residual vector)
∇J(w) = (2/n) Xᵀ(Xw − y)
update:  w := w − η·∇J(w) = w − η·(2/n)Xᵀ(Xw − y)
Closed-form (XᵀX)⁻¹XᵀyGradient descent
Best forsmall/medium, exactlarge data, many features, NNs
Costmatrix inverse (~n³)cheap iterations

Deep models have no closed form, so they rely on this gradient update.

7. When is the solution unique?

β̂ = (XᵀX)⁻¹Xᵀy needs XᵀX to be invertible — i.e. the feature columns are linearly independent (full column rank).

perfectly correlated / duplicate features → XᵀX singular → no unique solution (multicollinearity)

Fixes: drop redundant features or add regularization (ridge).

8. Projection view — the geometry

Geometrically, least squares projects y onto the column space of X (all reachable predictions):

ŷ = projection of y onto span(features)
residual e = y − ŷ  is orthogonal to that space

The optimality condition is exactly this orthogonality:

Xᵀ(y − Xβ̂) = 0     (residual ⟂ every feature)
9. Why least squares connects everything
  • Linear algebra: projection, column space, matrix inverse.
  • Optimization: minimize a convex squared-error objective.
  • Statistics: slope = Cov/Var; under noise assumptions OLS is the best linear unbiased estimator.
10. Why ridge regression exists GATE extra

Plain OLS (minimize RSS) can misbehave when features are correlated, there are many features, or it overfits — the coefficients blow up and become unstable.

symptom: tiny data changes → huge coefficient swings

Ridge regression tames this by adding a penalty on coefficient size.

11. OLS vs ridge objective (L2 penalty) GATE extra 
OLS:    minimize  RSS = Σ(yᵢ − ŷᵢ)²
Ridge:  minimize  RSS + λ Σ βⱼ²        (the λΣβ² term = L2 penalty)

Ridge doesn't only want low error — it also wants small coefficients. Large βⱼ ⇒ large penalty (β=10 → β²=100), so coefficients are pushed down.

12. Regularization = a power limiter GATE extra

Regularization adds a penalty so the model can't become too wild — it trades a little training fit for better generalization.

OLS   = unlimited power (can overfit)
Ridge = power limiter (more stable)
13. The λ (lambda) knob GATE extra 
λ = 0      → no penalty = ordinary OLS
small λ    → coefficients shrink a little
large λ     → strong shrinkage, simpler model
very large λ → coefficients → 0, model underfits

λ controls how much "discipline" the model has; it's tuned by cross-validation.

14. Coefficient shrinkage (ridge vs lasso) GATE extra

Ridge shrinks coefficients toward zero but rarely to exactly zero:

OLS   β = 20, −15, 8
Ridge β = 12,  −9, 5   (smaller, less sensitive)

Contrast: Lasso (L1 penalty) can set coefficients exactly to 0, doing feature selection — ridge does not.

15. Ridge, multicollinearity & the matrix form GATE extra

When features are highly correlated, XᵀX is nearly singular and OLS is unstable. Ridge adds λI to the diagonal, making the inverse well-behaved:

OLS:   β̂ = (XᵀX)⁻¹ Xᵀy
Ridge: β̂ = (XᵀX + λI)⁻¹ Xᵀy

The +λI guarantees an invertible matrix even with multicollinearity.

16. Ridge — pros, cons & when to use GATE extra
  • ✓ reduces overfitting, handles multicollinearity, stabilizes coefficients, improves generalization
  • ✗ no feature selection (coeffs don't hit 0), needs λ tuning, too-large λ underfits, less interpretable

Use it with many correlated features (house prices, medical measurements, ad-spend channels, pixel features).

17. The regularization landscape GATE extra

Regularization = adding a penalty/constraint so the model stays simple and generalizes. The main flavours:

  • L2 / Ridge — penalize Σβ² (shrinks coefficients)
  • L1 / Lasso — penalize Σ|β| (shrinks and zeros coefficients)
  • Elastic Net — mix of L1 + L2
  • For neural nets: dropout, early stopping, data augmentation, batch normalization
18. Lasso regression (L1) GATE extra

Lasso adds an L1 penalty — the sum of absolute coefficients:

minimize  RSS + λ Σ |βⱼ|

The absolute-value penalty is what lets coefficients hit exactly zero.

19. Lasso does feature selection GATE extra

Because L1 can drive weights to 0, lasso effectively drops useless features — an automatic feature selector giving a sparse, interpretable model.

ridge: β = 12, −9, 5   (all kept, just smaller)
lasso: β = 14,  0, 0   (only the useful feature survives)
20. Ridge vs Lasso GATE extra
Ridge (L2)Lasso (L1)

Elastic Net blends both (λ₁Σ|β| + λ₂Σβ²) — sparsity plus stability with correlated features.

21. Choosing the penalty & tuning λ GATE extra
  • Want a simpler, interpretable model / feature selection → Lasso.
  • Many correlated features, want stability → Ridge.
  • Both → Elastic Net.

Pick λ (a.k.a. α) by grid search with cross-validation — the value with the best validation score.

22. OLS assumptions & regression diagnostics GATE extra

OLS is only the best estimator (BLUE — Best Linear Unbiased Estimator) when a handful of assumptions hold. GATE loves testing whether you can recognise them:

  • Linearity: $y$ is linear in the parameters $\beta$ (not necessarily in $x$ — $x^2$ is fine).
  • Zero-mean errors: $E[e_i]=0$ — the model isn't systematically over- or under-predicting.
  • Homoscedasticity: $\operatorname{Var}(e_i)=\sigma^2$ is constant across all values of $x$ (errors don't fan out).
  • No autocorrelation: errors are uncorrelated with each other (a big deal for time-series data).
  • Error-free inputs: the $x$'s are measured without noise — only $y$ carries random error.
  • No perfect multicollinearity: no predictor is an exact linear combination of the others (else $X^\top X$ is singular).

Diagnosing violations — the residual plot:

plot residuals eᵢ = yᵢ − ŷᵢ  against ŷᵢ (or xᵢ)

random scatter around 0, constant spread → assumptions look fine
funnel / cone shape                     → heteroscedasticity (variance grows with x)
a curve or pattern                      → missed non-linearity (wrong model form)

Careful: $\sum e_i=0$ is an algebraic fact whenever the model has an intercept (it drops straight out of the normal equations) — it is not evidence that the zero-mean-error assumption holds for the true population errors.

23. OLS & outlier sensitivity GATE extra

Because OLS minimizes squared error, one far-away point contributes $e_i^2$ — squaring massively amplifies large residuals, so a single outlier can drag the whole fitted line toward it.

9 points fit ŷ = 2x closely; one outlier added far above the trend
→ the slope and intercept both shift noticeably to chase that one point

Outliers matter most when they also have high leverage — an extreme $x$ value combined with a large residual has outsized influence on $\hat\beta$ (an outlier at a "typical" $x$ pulls the line far less).

GATE-relevant fixes:

  • Inspect residual / scatter plots first — outliers are usually visible.
  • Robust regression (e.g. Huber loss, LAD/L1 regression) — penalizes large errors linearly instead of quadratically, so it isn't dominated by a few points.
  • RANSAC — repeatedly fits on random subsets and keeps the fit with the most inliers.
  • Winsorize/cap or remove points only with domain justification — don't drop outliers just because they're inconvenient.
24. Data imputation — handling missing values GATE extra

OLS (and most models) cannot fit rows with missing $x$ or $y$ values, so missing data must be handled first.

TechniqueIdeaWatch out for
Listwise deletiondrop any row with a missing valuewastes data; biased unless missingness is completely random
Mean / median imputationfill with the column's mean (or median, if skewed)shrinks variance, weakens correlations with other features
Mode imputationfill categorical values with the most frequent categorysame variance-shrinking issue
Regression imputationpredict the missing feature from the other featurescan overfit / understate uncertainty
kNN imputationaverage the value from the $k$ nearest complete rowsneeds scaled features and a distance metric (see the kNN topic)

GATE terminology for why data is missing:

MCAR — Missing Completely At Random: missingness unrelated to any value
MAR  — Missing At Random: missingness depends on other observed features
MNAR — Missing Not At Random: missingness depends on the missing value itself (hardest case)

Simple imputations (mean/median) are safest under MCAR; under MAR/MNAR they can quietly bias the fitted $\hat\beta$.

25. Multicollinearity — predictions can stay unique GATE extra

When features are perfectly collinear, $X^\top X$ is singular, so the coefficients $\hat\beta$ are not unique — infinitely many $\beta$ vectors fit equally well. A subtle but exam-favourite fact: the predictions $\hat y=X\hat\beta$ can still be unique even though $\hat\beta$ itself isn't.

x₂ = 2·x₁ exactly (perfectly collinear)
model: ŷ = β₀ + β₁x₁ + β₂x₂ = β₀ + β₁x₁ + β₂(2x₁) = β₀ + (β₁+2β₂)x₁

→ infinitely many (β₁, β₂) pairs give the same combined slope (β₁+2β₂)
→ same ŷ for every one of them, even though the individual β₁, β₂ differ

Why: the fitted value $\hat y$ only depends on $X\hat\beta$, which is the (unique) projection of $y$ onto the column space of $X$ — collinearity just means many different $\beta$ vectors land on the same point in that column space. So a singular $X^\top X$ breaks coefficient interpretability, not necessarily prediction quality.

26. L1 vs L2 — the non-differentiability difference GATE extra

Ridge (L2, $\lambda\sum w_j^2$) and Lasso (L1, $\lambda\sum|w_j|$) look similar but behave very differently right at $w=0$:

L2 penalty  w² :  smooth everywhere, derivative 2w → 0 smoothly as w→0
L1 penalty |w| :  has a SHARP CORNER (kink) at w=0 — not differentiable there

The derivative of $|w|$ is $+1$ for $w>0$ and $-1$ for $w<0$, but at exactly $w=0$ there's no single tangent slope — only a range of valid "subgradients" between $-1$ and $+1$. This corner is precisely why Lasso can push coefficients to exactly $0$ (feature selection): the flat-bottomed L1 penalty "catches" a coefficient at zero, whereas L2's smooth bowl only ever shrinks a coefficient toward zero without fully reaching it.

27. Gradient-descent update with an L2 penalty GATE extra

Ridge's loss adds $\lambda\|w\|^2$ to the usual squared-error loss $J(w)$. Differentiating the penalty term contributes an extra $2\lambda w$ to the gradient, giving the update rule:

$$ w := w-\eta\Big(\nabla J(w)+2\lambda w\Big) = w(1-2\eta\lambda)-\eta\nabla J(w) $$

Reading the rewritten form: every step first shrinks $w$ by the factor $(1-2\eta\lambda)$ (this is the "weight decay" you may have heard of in neural-network training), then applies the usual gradient step. Larger $\lambda$ ⇒ more shrinkage per step ⇒ smaller final weights.

28. Traps, practice & memory box
  • Least squares minimizes raw errors? No — squared errors (raw ones cancel).
  • (XᵀX)⁻¹ always exists? No — only with full-rank (independent) features.
  • Residuals are parallel to features? No — orthogonal at the optimum.

Q1. Cov(x,y)=4, Var(x)=8, x̄=10, ȳ=25 — slope & intercept?

β₁ = 4/8 = 0.5 ;  β₀ = 25 − 0.5·10 = 20

Q2. Why can't we solve OLS with two identical feature columns?

XᵀX is singular (rank-deficient) → no unique inverse
Memory: OLS = minimize ‖y−Xβ‖² → β̂=(XᵀX)⁻¹Xᵀy → ŷ = projection of y onto feature space.

6 Main formulas

Objective — minimize the sum of squared errors:

$$ \min_{\beta}\ \text{SSE}=\sum_{i=1}^{n}\big(y_i-\hat{y}_i\big)^2 $$

Normal equations — setting the derivative to zero gives the closed-form solution:

$$ X^{\mathsf T}X\,\beta = X^{\mathsf T}y \qquad\Rightarrow\qquad \beta=(X^{\mathsf T}X)^{-1}X^{\mathsf T}y $$

Ridge (L2) least squares — penalize large coefficients and stabilize a near-singular $X^{\mathsf T}X$:

$$ \min_{\beta}\ \text{RSS}+\lambda\textstyle\sum_j\beta_j^2 \qquad\Rightarrow\qquad \hat{\beta}=(X^{\mathsf T}X+\lambda I)^{-1}X^{\mathsf T}y $$

$\lambda=0$ recovers OLS; a larger $\lambda$ shrinks the coefficients (Lasso/L1 can push them exactly to 0).

7 Step-by-step method
  1. Write predictions $\hat{y}_i=\beta_0+\beta_1 x_i$ and the SSE $=\sum(y_i-\hat{y}_i)^2$.
  2. Take partial derivatives w.r.t. $\beta_0,\beta_1$ and set them to zero.
  3. Solve the resulting two equations (the normal equations) for the coefficients.
  4. In matrix form: $\beta=(X^{\mathsf T}X)^{-1}X^{\mathsf T}y$.
8 Worked example
✦ Why squared (not absolute)?

Squaring makes the SSE a smooth bowl in $(\beta_0,\beta_1)$, so setting derivatives to zero gives a single exact solution. For the points $(1,2),(2,4),(3,5),(4,4),(5,5)$ this yields $\hat{y}=2.2+0.6x$ — the same line the lab snaps to with Best fit.

9 GATE cheat sheet
⚡ High-value facts
  • Ridge: $\hat{\beta}=(X^{\mathsf T}X+\lambda I)^{-1}X^{\mathsf T}y$ — the $\lambda I$ fixes multicollinearity; shrinks but doesn't zero coefficients ($\lambda=0\Rightarrow$ OLS).
  • Least-squares minimizes $\sum e_i^2$ (not $\sum|e_i|$).
  • Closed form: $\beta=(X^{\mathsf T}X)^{-1}X^{\mathsf T}y$ (when $X^{\mathsf T}X$ is invertible).
  • At the optimum the residuals sum to zero and are uncorrelated with the predictors.
  • Squared error is convex ⇒ a unique global minimum (no local traps).
  • Gradient descent reaches the same solution iteratively (link to Optimization).
10 Common traps
✕ Watch out

Trap: "Least-squares minimizes absolute error." Reality: it minimizes squared error.

Trap: "It minimizes perpendicular distance." Reality: ordinary least squares uses vertical residuals.

Trap: "$(X^{\mathsf T}X)^{-1}$ always exists." Reality: it fails when features are linearly dependent (multicollinearity).
