k-Nearest Neighbors

8 questions. Tap any option — you'll see why it's right or wrong, and the correct one lights up.

Q1. kNN is a…
Clustering is k-means; kNN uses labels.
✅ Correct — it uses labeled neighbors to predict.
That's PCA's job.
kNN isn't a density model.
Q2. Does kNN have a training phase?
It just stores the data.
✅ Correct — lazy learner; work happens at prediction time.
No training either way.
Scaling is preprocessing, not training.
Q3. Neighbors (k=5) are {A,B,A,A,B}. Prediction?
A has the majority (3 vs 2).
✅ Correct — A appears 3 times, B 2 times.
5 votes can't tie between two classes.
It's well-defined: A.
Q4. Why use odd $k$ for two classes?
Speed is unaffected.
Not always more accurate.
✅ Correct — odd $k$ prevents tied votes.
Even $k$ is allowed (ties just need a rule).
Q5. As $k$ grows, the model becomes…
Small $k$ is high variance.
Small $k$ overfits, not large.
✅ Correct — bigger $k$ smooths the boundary (more bias).
It changes with $k$.
Q6. Before kNN, you should usually…
Labels are needed for classification.
✅ Correct — unscaled features distort the distance.
Sorting doesn't matter.
Scaling matters a lot.
Q7. kNN prediction cost per query is about…
It must look at all points.
Only with special index structures.
✅ Correct — basic kNN scans all $n$ training points.
One query is linear, not quadratic.
Q8. The usual kNN distance metric is…
Not a distance metric here.
Unrelated.
✅ Correct — Euclidean distance is the default.
Distance, not probability.
