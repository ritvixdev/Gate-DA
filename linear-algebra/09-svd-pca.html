Topic 9 · GATE weight: High Support

SVD & PCA

Every matrix — square or not — is secretly "rotate, stretch, rotate". That single fact powers dimensionality reduction across machine learning.

2 Why this topic exists

Diagonalization only works for some square matrices. The SVD works for any matrix and reveals its most important directions, ranked by strength. Keeping only the strongest few gives the best simple approximation — that's compression and PCA.

3 Game / visual imagination
🎮 Imagine it

Any transformation = rotate ($V^{\mathsf T}$) → stretch along clean axes ($\Sigma$) → rotate again ($U$). A circle always becomes an ellipse; the singular values $\sigma_i$ are the ellipse's axis lengths — the biggest one is the data's strongest direction.

4 Basic definitions
  • SVD: $A=U\Sigma V^{\mathsf T}$ for any $m\times n$ matrix.
  • $U$ ($m\times m$): orthogonal, left singular vectors. $V$ ($n\times n$): orthogonal, right singular vectors.
  • $\Sigma$: diagonal of singular values $\sigma_1\ge\sigma_2\ge\dots\ge0$.
  • PCA: finds the directions of maximum variance in centered data — the principal components.
📖 Study in Depth — SVD & PCA from zero, one idea per card
1. Why SVD is needed

Eigenvalue decomposition works mainly for square matrices. But real data is usually rectangular — e.g. a dataset X = 1000 samples × 20 features (1000×20).

SVD works for every matrix — square, rectangular, singular, rank-deficient — which is why it is called the crown jewel of linear algebra.

2. The main formula: A = UΣVᵀ 
A (m×n) = U Σ Vᵀ

U = m×m orthogonal     (UᵀU = I, U⁻¹ = Uᵀ)
Σ = m×n diagonal-like  (singular values)
V = n×n orthogonal     (VᵀV = I, V⁻¹ = Vᵀ)
3. Game imagination: rotate → stretch → rotate

Reading Ax = UΣVᵀx right to left, every matrix is just three simple steps:

Vᵀ → rotate/reflect the input into clean axes
Σ  → stretch each axis by a singular value
U  → rotate/reflect into the output space

SVD = rotate → stretch → rotate. The best mental model.

4. Singular values

Σ holds the singular values σ₁ ≥ σ₂ ≥ … ≥ 0 — always non-negative, sorted largest first.

σ₁ = strongest stretch direction
σ₂ = next strongest, …
5. The shape of Σ

Σ has the same shape as A, with singular values on the diagonal positions:

A is 3×2:  Σ = [σ₁ 0; 0 σ₂; 0 0]
A is 2×3:  Σ = [σ₁ 0 0; 0 σ₂ 0]
6. U and V (singular vectors) 
V columns = right singular vectors → live in INPUT space (n-dim), V is n×n
U columns = left  singular vectors → live in OUTPUT space (m-dim), U is m×m
Σ connects them by stretching
7. The core relationship 
A vᵢ = σᵢ uᵢ        (input direction vᵢ → output direction uᵢ, scaled by σᵢ)
Aᵀuᵢ = σᵢ vᵢ

Clean input direction vᵢ maps to clean output direction uᵢ, stretched by σᵢ.

8. Relationship with eigenvalues (key) 
singular values:  σᵢ = √( eigenvalue of AᵀA )
eigenvalues of AᵀA = σᵢ²

V = eigenvectors of AᵀA
U = eigenvectors of AAᵀ

Memory: AᵀA gives V and σ²; AAᵀ gives U and σ².

9. Why AᵀA's eigenvalues are ≥ 0 
xᵀAᵀAx = (Ax)ᵀ(Ax) = ||Ax||² ≥ 0

So AᵀA is positive semidefinite — its eigenvalues are non-negative, which is exactly why singular values are real and ≥ 0.

10. Rank & nullity from singular values 
rank(A) = number of NON-ZERO singular values
nullity(A) = n − rank   (zero singular directions = null space)
σ = 5, 3, 0, 0  →  rank 2

Zero singular values mark the directions the matrix destroys.

11. SVD examples

Diagonal matrix (already aligned):

A = [3 0; 0 2]  →  U = I, Σ = [3 0; 0 2], V = I,  σ = 3, 2

Negative diagonal — singular values are still positive:

A = [−3 0; 0 2]  →  eigenvalues −3, 2  but  singular values 3, 2
(the flip goes into U or V; Σ only stretches)

13. Compact (reduced) SVD GATE extra

If rank = r, keep only the non-zero part:

A = Uᵣ Σᵣ Vᵣᵀ      (Uᵣ: m×r, Σᵣ: r×r, Vᵣ: n×r)

This is what compression and PCA actually use.

14. Outer-product form GATE extra 
A = σ₁u₁v₁ᵀ + σ₂u₂v₂ᵀ + … + σᵣuᵣvᵣᵀ

Each σᵢuᵢvᵢᵀ is a rank-1 pattern; A is a sum of patterns, the first being most important. This is the basis of low-rank approximation.

15. Low-rank approximation GATE extra 
Aₖ = σ₁u₁v₁ᵀ + … + σₖuₖvₖᵀ      (keep top k, discard the rest)

Since σ₁ ≥ σ₂ ≥ …, the first few terms hold most of the information.

16. Why low-rank approximation works GATE extra

An image is a matrix of pixel intensities. SVD splits it into layers (strongest structure first). Keeping the top ~20 singular values can still look clear.

So SVD compresses by keeping important directions — used in image compression, noise reduction, recommender systems, latent semantic analysis, PCA.

17. Best rank-k approximation (Eckart–Young) GATE extra

Aₖ is the best rank-k approximation of A in Frobenius norm — it minimizes ||A − Aₖ||_F over all rank-k matrices.

||A||_F = √(sum of squares of entries)
||A||_F² = σ₁² + σ₂² + … + σᵣ²    (singular values = the matrix's "energy")
18. Energy / variance captured GATE extra 
energy kept by top k = (σ₁² + … + σₖ²) / (σ₁² + … + σᵣ²)
σ = 10, 5, 1 → total 126; top 2 = 125 → 125/126 ≈ 99.2%

A rank-2 approximation can keep almost all the information.

19. Pseudo-inverse via SVD 
A = UΣVᵀ  →  A⁺ = V Σ⁺ Uᵀ
Σ⁺: replace each non-zero σᵢ with 1/σᵢ, transpose the shape, keep zeros

The pseudo-inverse works when A is rectangular, singular, or rank-deficient — where a normal inverse does not exist.

20. Condition number GATE extra 
κ(A) = σmax / σmin   (= σ₁ / σᵣ for full rank)
large κ → ill-conditioned (small input error → large output error)
κ ≈ 1  → well-conditioned;  orthogonal matrix → all σ = 1 → κ = 1 (best)
21. What is PCA?

Principal Component Analysis finds the directions where data varies the most — the best new axes for the data. They are orthogonal and ordered by importance:

PC1 = max variance ;  PC2 = max remaining variance ⊥ PC1 ;  …
22. PCA game imagination

A 2D data cloud stretched diagonally doesn't line up with the x/y axes. PCA rotates the coordinate system to match the data:

PC1 = direction of maximum spread
PC2 = perpendicular direction of next spread

PCA = finding the natural directions of the data.

23. PCA & variance

Variance = spread. PCA keeps high-variance directions and discards low-variance ones, because high variance usually carries the information and low variance is often noise.

24. PCA step by step

Data X is n samples × d features.

  1. Center: X̄ = X − mean(X) (each feature mean 0) — PCA needs centered data.
  2. Covariance: C = (1/n) X̄ᵀX̄ (a d×d symmetric matrix).
  3. Eigendecompose: Cv = λv (real eigenvalues, orthonormal eigenvectors).
  4. Sort eigenvalues descending: λ₁ ≥ λ₂ ≥ …; v₁, v₂, … are the principal components.
  5. Project: keep top k → Vₖ, then Z = X̄Vₖ (n×k reduced data).
25. PCA & SVD connection

Instead of forming the covariance matrix, apply SVD directly to the centered data:

X̄ = UΣVᵀ  ⇒  X̄ᵀX̄ = VΣᵀΣVᵀ  ⇒  C = (1/n)·VΣᵀΣVᵀ
principal components = columns of V
covariance eigenvalues = σᵢ²/n   (or σᵢ²/(n−1) for sample covariance)
26. PCA interpretation (explained variance)

Each eigenvalue = variance along its principal component.

explained variance ratio = λᵢ / (λ₁ + λ₂ + … + λd)
λ = 50, 10, 1 → PC1 ≈ 81.97%,  PC1+PC2 ≈ 98.36%

So 2 components can preserve almost all the information.

27. PCA for dimensionality reduction

If 100 features have, say, 10 components explaining 95% variance, reduce 100 → 10 dimensions.

benefits: less storage, faster training, less noise, easier visualization, less multicollinearity
downside: components are combinations of features → less interpretable
28. PCA conditions (centering & scaling)

PCA must be done on centered data. If features have very different scales (income 0–10,00,000 vs age 0–100), the large-scale feature dominates — so standardize first:

standardize = subtract mean, divide by standard deviation
29. Covariance matrix properties 
C = (1/n) X̄ᵀX̄  is symmetric and positive semidefinite
vᵀCv = (1/n)||X̄v||² ≥ 0  →  eigenvalues ≥ 0

So PCA eigenvalues are variances and can never be negative; eigenvectors can be chosen orthonormal.

31. SVD & least squares

When Ax = b has no exact solution, the least-squares answer is:

x̂ = A⁺b = VΣ⁺Uᵀ b

This works even for rectangular / rank-deficient A. For full column rank it matches the normal-equations answer x̂ = (AᵀA)⁻¹Aᵀb from the Orthogonality topic.

32. SVD & determinant GATE extra 
square A:  |det(A)| = product of singular values

Any zero singular value ⇒ det = 0 (singular). Singular values give the magnitude of the volume scaling, not its sign.

33. SVD vs eigenvalues — careful! 
A = [−2]:  eigenvalue = −2,  singular value = 2

Singular values are non-negative stretch amounts; eigenvalues carry sign/complex behaviour. They align only for symmetric positive-semidefinite matrices (where eigenvalues are ≥ 0).

35. Key formulas
  • A = UΣVᵀ;   σᵢ = √λᵢ(AᵀA);   rank = #non-zero σ.
  • ||A||_F² = σ₁² + … + σᵣ²;   Aₖ = Σ_{i=1..k} σᵢuᵢvᵢᵀ.
  • energy = (σ₁²+…+σₖ²)/(σ₁²+…+σᵣ²);   A⁺ = VΣ⁺Uᵀ;   κ = σmax/σmin.
  • PCA: C = (1/n)X̄ᵀX̄, Z = X̄Vₖ, λᵢ = σᵢ²/n (or /(n−1)).
36. Dimensionality reduction — the landscape

Too many features cause the curse of dimensionality: data gets sparse, distances lose meaning, models overfit, and plots are impossible beyond 3-D. Dimensionality reduction keeps the useful information in fewer features.

feature selection  = keep a subset of original features
feature extraction = build new combined features (PCA, autoencoders); LDA uses labels

Benefits: faster training, less storage, easier visualization, reduced overfitting. (PCA — the main extraction method — is detailed in the cards above.)

37. LDA — supervised dimensionality reduction GATE extra

Linear Discriminant Analysis finds the direction(s) that best separate classes — used for classification, dimensionality reduction and feature extraction.

project many features onto a new axis where the classes barely overlap

Unlike PCA it uses the labels, so it's supervised.

38. Fisher's criterion — the goal GATE extra

LDA picks the projection direction w that simultaneously:

maximize  (distance between class means)²
minimize  (spread within each class)
J(w) = between-class scatter / within-class scatter   → maximize

Classes end up far apart and individually tight.

40. How many LDA components? (C − 1) GATE extra 
max LDA dimensions = (number of classes) − 1
2 classes → 1 component · 3 → 2 · 5 → 4

A common GATE fact — LDA can't produce more than C−1 useful directions.

41. LDA assumptions GATE extra
  • features ≈ normally distributed within each class
  • equal covariance across classes (this makes the boundary linear)
  • observations independent

If classes are non-linear, kernel SVM may do better.

42. LDA — pros, cons, uses GATE extra
  • ✓ simple, interpretable, great for dimensionality reduction, good on small/medium data, curbs overfitting
  • ✗ assumes linear separation, normality & equal covariance; sensitive to outliers

Uses: face recognition, image/text classification, bioinformatics, medical diagnosis.

43. Common GATE traps
  • SVD works only for square matrices? No — every m×n.
  • Singular values can be negative? No — always ≥ 0.
  • Singular values are eigenvalues of A? No — they are √(eigenvalues of AᵀA).
  • #non-zero σ = determinant? No — it equals the rank.
  • PCA on raw uncentered data? No — center (and often standardize) first.
  • First PC from the smallest eigenvalue? No — the largest. PCs are orthogonal; eigenvalues = variance. (All true.)

45. Cheatsheet
  • A = UΣVᵀ for ANY matrix; U, V orthogonal; σ ≥ 0 sorted descending.
  • V = eigenvectors of AᵀA; U = of AAᵀ; σ = √λ(AᵀA); rank = #non-zero σ.
  • Best rank-k = top-k σ terms; energy = ratio of σ²; A⁺ = VΣ⁺Uᵀ; κ = σmax/σmin.
  • PCA = SVD/eigendecomposition of centered data; PCs = columns of V; eigenvalues = variance.
  • Center (and standardize) before PCA; covariance is symmetric PSD; largest eigenvalue = PC1.
46. Best mental model

SVD = a universal matrix X-ray. It reveals which input directions matter, how strongly they stretch, which output directions they become, and which are destroyed.

PCA = SVD applied to data to find the most important directions: which feature-combination has maximum spread, the next, and how few directions still preserve most information.

Eigen/eigenvectors → square matrices
SVD               → ANY matrix
PCA               → uses SVD/eigenvectors to reduce data

6 Main formulas
$$ A = U\,\Sigma\,V^{\mathsf T} $$
$$ \sigma_i=\sqrt{\lambda_i(A^{\mathsf T}A)},\qquad V:\text{eigenvectors of }A^{\mathsf T}A,\qquad U:\text{eigenvectors of }AA^{\mathsf T} $$
$$ A \approx \sum_{i=1}^{k}\sigma_i\,u_i v_i^{\mathsf T} \quad(\text{best rank-}k\text{ approximation}) $$
7 Step-by-step: compute an SVD
  1. Form $A^{\mathsf T}A$ (symmetric, positive semi-definite).
  2. Find its eigenvalues $\lambda_i\ge0$ and eigenvectors → those eigenvectors are $V$.
  3. Singular values $\sigma_i=\sqrt{\lambda_i}$, placed (largest first) on $\Sigma$.
  4. Get each $u_i=\dfrac{1}{\sigma_i}A v_i$ → those form $U$.
  5. Assemble $A=U\Sigma V^{\mathsf T}$.
8 Worked example (idea)
✦ Singular values

For $A=\begin{bmatrix}3&0\\0&-2\end{bmatrix}$: $A^{\mathsf T}A=\begin{bmatrix}9&0\\0&4\end{bmatrix}$, eigenvalues $9,4$.

So singular values are $\sigma_1=3,\ \sigma_2=2$ (always non-negative — note the $-2$ became $2$).

9 GATE cheat sheet
⚡ High-value facts
  • Singular values are $\sqrt{\text{eigenvalues of }A^{\mathsf T}A}$ — always real and $\ge0$.
  • Number of non-zero singular values $=\operatorname{rank}(A)$.
  • $U,V$ are orthogonal; $\Sigma$ is diagonal.
  • PCA = SVD of the centered data matrix; variance explained $=\sigma_i^2/(n-1)$.
10 Common traps
✕ Watch out

Trap: "Singular values = eigenvalues of $A$." Reality: only for symmetric PSD matrices; in general $\sigma_i=\sqrt{\lambda_i(A^{\mathsf T}A)}$.

Trap: "Singular values can be negative." Reality: never — they are $\ge0$ by definition.

Trap: "Only square matrices have an SVD." Reality: every matrix does.

11 Connections with other topics
  • Eigenvalues: SVD is built from eigenvalues of $A^{\mathsf T}A$ and $AA^{\mathsf T}$.
  • Orthogonality: $U$ and $V$ are orthogonal matrices.
  • Rank: number of non-zero singular values.
  • Diagonalization: SVD is the "works-for-everything" generalization.
